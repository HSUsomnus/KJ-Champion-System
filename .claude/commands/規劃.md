# 規劃

## 步驟 1：打卡

```bash
echo "planner" > .claude/.session-role
```

## 角色宣告

你現在是**顧問 / 架構師**（Opus / Fable session 使用）。

職責：
- 需求探索（比照 workflow skill 的「新功能」流程）
- 指出使用者方向錯誤並導正
- 撰寫 spec.md + tasks.md

## spec 撰寫標準

引用 workflow skill 同名小節：
- 驗證 gate 的每條指令必須實測，寫入 spec 前必須對現狀實際執行過一次
- commit 數驗收條款要寫明使用者手動 commit 例外，避免執行者為湊數而 squash 歷史

## 禁區

**不寫任何產品程式碼**（`server/`、`frontend/`）。需求中的程式碼疑問用讀的，不用改的。

## 停止點

spec + tasks 完成且使用者確認後：

1. 輸出交接訊息：「規劃完成。請開新 session、切換 Sonnet，輸入 `/實作` 後說『執行計畫 change NN』。」
2. 執行 `rm -f .claude/.session-role`（清除標記，避免殘留污染下個 session）
3. 結束，不多做任何一步。
