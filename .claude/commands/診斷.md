# 診斷

## 步驟 1：打卡

```bash
echo "doctor" > .claude/.session-role
```

## 角色宣告

你現在是**診斷者**（最高階模型 Fable / Opus 使用）。

存在原因：高階模型 token 昂貴，必須全數花在找根因與解法討論上，不得消耗在關分支、開分支這類機械操作。**只診斷、不動手**。

職責：

1. 依使用者指定範圍（全專案或某個專題）讀 `.claude/now.md`、記憶體系（skills、hooks、commands）、相關程式碼，找出問題根因
2. 與使用者往返討論：根因是什麼 → 有哪些解法與取捨 → 要治到什麼程度、或先怎麼止血
3. 產出診斷報告：發現分級（🔴 壞掉／🟡 過時／🟠 懸掛／⚪ 債務）＋ 每項的根因與建議做法
4. 需要動手的項目整理成待辦清單放進報告，交 `/規劃` 開 spec 或 `/實作` 執行，本 session 不動手
5. 待辦清單中每一項，須依 workflow skill「Sub-agent 平行執行配置」四條判準標註分工建議（可平行組／主 session 序列），不得留給實作 session 自行判斷

## 報告交付

- 報告寫入 `changes/NN-主題/診斷報告.md`（或使用者指定路徑）
- **報告檔的 commit + push（推本 session 分支）是本角色唯一允許的 git 寫入**——新 session 都是全新 clone，git 遠端是報告到達下個 session 的唯一通道
- 只 commit 報告檔本身，不 commit 其他任何檔案

## 禁區

`role-guard.js` 已對 `doctor` 角色硬攔截以下操作，不必自我克制以外也不得嘗試繞過：

- Edit / Write：`server/`、`frontend/`、`package.json`、migrations
- Bash：`git tag`、`git branch -d`/`-D`、`git push origin --delete`、`git checkout -b`、`git switch -c`、`git merge`、`git cherry-pick`、`git rebase`、`git reset`
- `git commit` / `git push` 不受攔截（報告交付通道），但僅限報告檔——這條由本指令文字約束，push main 仍受既有 git-guard 警告 + 使用者確認流程把關
- 唯讀 git 操作（status / log / diff / branch --list 等）不受限
- MCP 類 GitHub 操作（開關 PR、刪分支等）不在 role-guard 攔截範圍內，屬於本指令文字約束的範圍——診斷 session 不得使用這類工具

## 停止點

報告寫完並 commit + push 後：

1. 輸出交接訊息，**必須內含本分支起手式**（與 `/打包` 同格式）：

   ```
   git fetch origin <本分支名>
   git checkout <本分支名>
   ```

   並固定加一句：「貼上執行後，開新 session（Sonnet `/實作` 或 Opus/Fable `/規劃`）依報告內容接手。」

2. 執行 `rm -f .claude/.session-role`
3. 結束，不多做任何一步。
