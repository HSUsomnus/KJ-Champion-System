# 子代理指令

執行以下步驟：

## 步驟 1：掃描未合併分支

```bash
git branch --list 'm_b_*' --no-merged main
```

列出所有未合併到 main 的 `m_b_*` 分支。若無分支則告知使用者並結束。

## 步驟 2：建立 Worktree

對每個未合併分支，檢查 `.claude/worktrees/` 是否已有對應 worktree：
- 分支名 `m_b_XXX` → worktree 路徑 `.claude/worktrees/XXX`
- 若已存在則跳過，若不存在則執行 `git worktree add`

## 步驟 3：觸發 Hook 開啟 VS Code

所有 worktree 就緒後，執行以下指令觸發 PostToolUse hook 自動開啟所有 VS Code 視窗：

```bash
echo "--open-vscode-worktrees"
```

這會觸發 `.claude/hooks/open-worktree-vscode.js`，自動為每個 worktree 開啟獨立的 VS Code 視窗。

## 步驟 4：回報結果

列出表格顯示所有子代理狀態：

| 分支 | Worktree 路徑 | VS Code |
|------|--------------|---------|
| m_b_XXX | .claude/worktrees/XXX | ✅ 已開啟 |

告知使用者：「所有子代理視窗已開啟，切到各視窗對 Claude 說『繼續』即可接續開發。」
