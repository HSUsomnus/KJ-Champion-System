# 打包

## 定位

context 快滿、或要換裝置/換模型續作時，把「git 和檔案讀不出來的工作狀態」落盤推遠端。

⛔ /打包 **不呼叫 `/compact`、不壓縮 context**——如果只是想在**同一個 session** 繼續，
用內建 `/compact` 即可，不需要 /打包。/打包 是**跨 session 交接**：寫交接檔 → 新對話讀回，
兩者是不同機制，不要混用。

## 步驟 1：更新 `.claude/now.md`

依既有職責更新：當前 change、新發現的地雷、環境特殊狀態。

## 步驟 2：寫 `.claude/handoff.md`（交接檔）

固定六段，缺一不可：

1. **打包時間 + session 角色**（planner / engineer / 無，讀 `.claude/.session-role`）
2. **當前任務**：change 編號、分支、目標一句話
3. **已完成 / 進行中 / 下一步**（各條列）
4. **工作區狀態**：`git status --short` 的輸出貼上，附處置建議（哪些該 commit、哪些該丟）
5. **關鍵決策與注意事項**：本 session 內做過、但還沒進 spec/context 的判斷
6. **給 /繼續 的第一個動作**：明確一句（例：「跑 Phase 3 gate 後繼續 task 3.2」）

## 步驟 3：commit + push（分支分流）

- 在 `m_b_*` / `claude/*` 分支 → 功能分支本來就必推遠端，**直接 commit + push**
  （`.claude/handoff.md`，若 now.md 同分支一併 commit）。
- 在 `main` 分支 → 屬規則類直推，但 push main 需使用者明確確認：
  列出即將 commit 的檔案清單，等使用者確認後才 push；**未確認就停在 commit，不 push**。

## 步驟 4：回報收尾

回報**必附「下個 session 起手式」code block**（使用者直接複製貼進新對話執行）：

```
git fetch origin <工作分支>
git checkout <工作分支>
```

並固定加一句：「貼上執行後，輸入 `/繼續`。」

這是使用者跨 session 的唯一入口，不可省略。
