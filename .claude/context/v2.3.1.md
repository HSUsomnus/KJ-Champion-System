# v2.3.1 Context

## 版本摘要

hotfix — `server.js` `public/` 不存在時 GET / ENOENT → 500

## 問題根因

v2.0.0 刪除 `public/`（前端改由 Cloudflare Pages 服務）後，`server.js` 仍有舊邏輯：

```js
const publicPath = useReactFrontend ? reactDistPath : path.join(__dirname, '../public');
app.use(express.static(publicPath));       // 目錄不存在，express.static 本身不 crash
app.get('/', (req, res) => {
  res.sendFile(path.join(publicPath, 'index.html')); // ← ENOENT crash
});
```

prod Zeabur 環境：`USE_REACT_FRONTEND` 未設 1、`frontend/dist` 不在容器內
→ `publicPath = '../public'`（已刪除）
→ 任何 `GET /` 請求 → `ENOENT stat '/src/public/index.html'` → 500

同時 `favicon.ico` 有 hardcode `'../public/images/logo.png'` fallback，同樣不存在。

## 修法（server/server.js）

加一行：`const publicExists = fs.existsSync(publicPath);`

三處改為條件式：
1. `express.static` → `if (publicExists) app.use(express.static(publicPath))`
2. `favicon.ico` → 移除 fallbackFavicon hardcode，改 `if (fs.existsSync(faviconPath)) sendFile else 404`
3. `GET /` → `if (publicExists && fs.existsSync(indexPath)) sendFile else res.json({ status: 'ok', service: 'kj-champion-api' })`
