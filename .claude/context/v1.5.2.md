# v1.5.2 上下文記錄

**版本**：v1.5.2
**日期**：2026-03-18
**分支**：staging

---

## 版本目標

建立 staging 分支，將部署架構從 Vercel（前後端統一）拆分為：
- **後端**：Zeabur（Node.js 容器，`staging` 分支監聽）
- **前端**：Cloudflare Pages（React + Vite + PWA，`staging` 分支監聽）

此版本完成 Task 1（分支與後端設定檔建立），後續 Task 2~9 待後續版本完成。

---

## 本版變更

### 新增檔案

| 檔案 | 說明 |
|------|------|
| `zbpack.json` | Zeabur 部署設定（`nodejs_version: "18"`，`start_command: "node server/server.js"`） |
| `.github/workflows/keep-supabase-alive.yml` | GitHub Actions：定期 ping Supabase 防休眠 |
| `.github/workflows/monthly-backup.yml` | GitHub Actions：每月自動備份資料庫 |
| `scripts/backup-db.js` | 手動資料庫備份腳本（pg_dump） |
| `scripts/restore-db.js` | 資料庫還原腳本 |

### 修改檔案

#### `env.example`
- `DATABASE_URL` 說明從「Supabase PostgreSQL」改為「Zeabur PostgreSQL 設定」
- `APP_URL` 已為 Zeabur 格式（`https://your-app.zeabur.app`）
- `FRONTEND_URL` 已為 Cloudflare Pages 格式（`https://your-app.pages.dev`）

#### `.gitignore`
- 已加入 `!zbpack.json` 例外（因 `*.json` 規則會擋住 zbpack.json）

#### `server/config/db.js`
- 新增 Serverless / 容器模式判斷（`process.env.VERCEL`）
- Serverless 模式：`max: 2`，`idleTimeoutMillis: 10000`（防止 Supabase 連線超限）
- 容器/本機模式：`max: 10`，`idleTimeoutMillis: 30000`

#### `server/server.js`
- CORS 從全開（`cors()`）改為白名單模式
- 白名單：`localhost:8080`、`localhost:3000`、`APP_URL`、`FRONTEND_URL`
- 加入 `credentials: true` 允許帶 Cookie / Authorization header
- 啟動時 log 出 CORS 允許來源，方便 debug

#### `openspec/changes/split-deploy-cloudflare-zeabur/tasks.md`
- Task 1.1~1.4 標記為完成

## 架構說明（目標狀態）

```
staging 分支：
  後端 → Zeabur (Node.js container)
          └─ 連接 Zeabur PostgreSQL（staging 專用，與 Supabase 正式庫隔離）
  前端 → Cloudflare Pages (React + Vite + PWA)
          └─ /api/* 透過 _redirects proxy 至 Zeabur 後端

main 分支（正式，暫不動）：
  後端 → Vercel Serverless (api/index.js)
          └─ 連接 Supabase PostgreSQL
  前端 → Vercel Static (public/)
```
