## 做了什麼

### 修復 Vercel 部署 manifest.json 404 問題

**問題**：`line-liff-calendar.vercel.app` 上 `/manifest.json` 回傳 404，Service Worker 多次嘗試快取失敗。

**原因**：`vercel.json` 的 `@vercel/node` builder 預設不打包靜態資源目錄，只打包 JS 依賴。`public/manifest.json` 雖然在 git 中，但 Vercel 部署時未被包含進 serverless function bundle。

**修復**：在 `vercel.json` builds 加入 `includeFiles` 設定，明確告知 Vercel 打包 `public/**` 和 `server/**`：

```json
"config": {
  "includeFiles": ["public/**", "server/**"]
}
```

## 改動的檔案

| 檔案 | 改動 |
|------|------|
| `vercel.json` | builds config 加入 `includeFiles: ["public/**", "server/**"]` |
| `CHANGELOG.md` | 新增 v1.5.2 版本記錄 |

## 注意事項

- 此修復需要 Vercel 重新部署才能生效
- 若仍有問題，可考慮改用 Vercel 靜態資源 (`static` builder) 服務 `public/` 目錄
