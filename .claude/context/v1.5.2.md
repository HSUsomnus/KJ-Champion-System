# v1.5.2 上下文

## 版本資訊
- 日期：2026-03-18
- 分支：main
- 目標：Task 2e（雙寫備份服務）完成

## 本版主要變更

### 新增 `server/services/dualWriteService.js`
- 實作 `dualWrite(primaryFn, backupFn)`：主庫失敗直接拋錯，備份庫失敗 console.warn（fire-and-forget）
- 讀取 `DUAL_WRITE_ENABLED` 環境變數，`false` 時直接跑 primaryFn
- 備份連線池加入 `ssl: { rejectUnauthorized: false }`（Supabase 需要 SSL，這是 2e.6 失敗的根本原因）
- 匯出 `dualWrite`、`backupQuery`、`backupGetClient`、`isEnabled`

### 修改 `server/services/eventDbService.js`
- 引入 dualWriteService
- `upsertEvents`：重構為 `_upsertEventsToPool(getClientFn, events)` 共用邏輯，透過 dualWrite 同時備份
- `deleteEventsNotIn`：透過 dualWrite 同時備份刪除
- `deleteEventById`：透過 dualWrite 同時備份刪除

### 修改 `server/services/memberDbService.js`
- 引入 dualWriteService
- `createMember`、`updateMember`、`updateMemberRole`、`updateFinancialAmount` 均改用 dualWrite 包裝

### 修改 `env.example`
- 新增 `DUAL_WRITE_ENABLED` 和 `SUPABASE_BACKUP_URL` 說明

## 背景
Task 2d 已完成（正式 Vercel DATABASE_URL 切換至 Zeabur PostgreSQL）。
Task 2e.1–2e.3 的程式碼原本寫在 staging 分支但未 commit，導致 2e.5 Redeploy 沒有實際部署雙寫程式碼。
本版將雙寫程式碼正確放到 main 分支，讓正式 Vercel 生效。

## 環境變數（需在 Vercel 正式後端設定）
- `DUAL_WRITE_ENABLED=true`
- `SUPABASE_BACKUP_URL=postgresql://postgres:<password>@db.<project>.supabase.co:5432/postgres`
