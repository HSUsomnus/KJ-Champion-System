# v2.0.3 — _worker.js 依 Pages 網址自動路由後端

**Tag**: v2.0.3
**Date**: 2026-04-12
**Type**: hotfix
**Branch flow**: main → hotfix/worker-dev-backend-routing → main

## 背景

`frontend/public/_worker.js` 的 `ZEABUR_BACKEND` 常數寫死指向正式後端 `kj-champion-system.zeabur.app`。當有兩個 Cloudflare Pages 部署時（正式 `kj-champion-system.pages.dev` 和 dev `kjcs-dev.pages.dev`），dev 站的所有 `/api/*` 請求也被轉到正式後端。

## 症狀

手機在 `https://kjcs-dev.pages.dev/?eruda=1` 透過 Eruda Console 測試 dev 專屬的新 API：

| 測試 | 實際結果 | 根本原因 |
|------|---------|---------|
| GET `/api/line/agenda-settings` | 404「找不到請求的資源」 | _worker.js 把請求轉到正式後端，該後端還沒部署新 code |
| POST `/api/line/push-daily-agenda` | 500「CORS 不允許來源: https://kjcs-dev.pages.dev」 | 正式後端 CORS 白名單不含 kjcs-dev.pages.dev |
| PUT `/api/line/agenda-settings` | 500「CORS 不允許來源: https://kjcs-dev.pages.dev」 | 同上 |

## 修復

在 `frontend/public/_worker.js` 新增 `resolveBackend(hostname)`：

```javascript
const ZEABUR_BACKEND_PROD = 'https://kj-champion-system.zeabur.app';
const ZEABUR_BACKEND_DEV = 'https://kj-champion-system-dev.zeabur.app';

const resolveBackend = (hostname) => {
  if (hostname === 'kjcs-dev.pages.dev' || hostname.startsWith('kjcs-dev')) {
    return ZEABUR_BACKEND_DEV;
  }
  return ZEABUR_BACKEND_PROD;
};
```

Worker 內部所有引用 `ZEABUR_BACKEND` 的地方改用 `backend = resolveBackend(url.hostname)`。

## 影響

- ✅ 修復 `kjcs-dev.pages.dev` 的所有 API 呼叫（從此正確走 dev 後端）
- ✅ 對正式站 `kj-champion-system.pages.dev` **完全無影響**（仍走 `ZEABUR_BACKEND_PROD`）
- ✅ OAuth redirect 重寫邏輯同步更新為 `backend`（原本也是寫死 `ZEABUR_BACKEND`）
