## 關鍵設計決策

**Financial.jsx 保留不動**：`/financial?userId=xxx`（從 MemberDetail 查看他人財力）仍使用 Financial.jsx，未整合進 Profile.jsx。原因：Profile.jsx 只顯示自己的財力，他人財力是不同語境（傳入 userId param），合併會讓邏輯複雜化。

**財力 tab lazy 載入**：Profile.jsx 財力 tab 用 `financialLoaded` state + `useEffect` 在 `activeTab === 'financial'` 第一次切換時才打 API，避免進頁面就全部打三個 API。

**三個隱藏 toggle 實際生效**：原始 Profile.jsx 和 Financial.jsx 的 hideProfile / hideFinancial / hideDocuments state 存在但從未用於條件渲染（bug）。新版修正，toggle 開關確實隱藏對應區塊。

**MemberDetail tab key 不變**：TABS 改為物件陣列 `{ key, label }` 後，tab state 仍存 key string（'資料'/'數據'/'財力'），內容條件 `tab === '資料'` 等不需改，只有 tab 渲染改為 `t.label`。

## 學習日誌

### 問題 1：feature branch 未推就 merge dev

**遇到的問題**：push 到 dev 後 Cloudflare build 成功，但 UI 沒有變化。  
**試過的方法**：清 Service Worker、清快取，UI 仍舊。  
**最終解法**：發現 dev merge 用的是 `git merge origin/m_b_用戶資料整合`，但 feature branch 的 commits 還沒 push 到 remote，所以 merge 到的是舊的 remote HEAD（只有初始 branch 建立），不含任何程式碼變更。`git push origin m_b_用戶資料整合` 後重新 merge dev → push → 成功。  
**根本原因**：feature branch 本機有 commits 但未推 remote，remote HEAD 落後，merge remote branch 等於 merge 舊版本。

### 問題 2：SidebarNav 出現兩個「用戶資料」

**遇到的問題**：側邊欄同時出現兩個「用戶資料」入口。  
**根本原因**：誤把 `{ label: '用戶資料', path: '/profile' }` 加到 `NAV_ITEMS`，但 SidebarNav 底部已有固定的用戶按鈕（`handleProfile → navigate('/profile')`），兩者重複。  
**解法**：完全移除 NAV_ITEMS 新增的項目，只保留底部用戶按鈕，label 改為「用戶資料」即可。
