# v2.0.4 — 新增行程救援：靜默無反應 + 誤跳未儲存警告

**Tag**: v2.0.4
**Date**: 2026-04-13
**Type**: hotfix
**Branch flow**: main → hotfix/add-event-save-no-feedback → main

## 背景

使用者回報「新增行程」頁點紅色 FAB「確認」後**完全沒反應**。進一步測試發現即便 API 成功，回程還會誤跳「尚未儲存資料，確認離開？」警告。兩個 UX bug 疊加。

## 症狀

1. 在 `/add-event` 填部分欄位（或沒填）按「確認」→ 頁面無任何回饋
2. 填齊欄位按「確認」→ API 成功 → 出現分享詢問 → 點取消或確認分享後，仍跳「尚未儲存資料，確認離開？」

## 根因

### Bug 1：靜默 return 無回饋（AddEvent.jsx）

`handleConfirm` 原本：

```js
if (!form.title || !form.date) return  // ← 靜默退出，使用者不知道哪裡沒填
```

FAB 按鈕位於 `<form>` 外部，HTML 原生 `required` 不會觸發。使用者漏填時只會看到「完全沒反應」。

### Bug 2：useLeaveGuard 時序競態（ConfirmLeaveDialog.jsx）

原本：

```js
const [saved, setSaved] = useState(false)
const blocker = useBlocker(!saved)
```

`useState` 更新非同步，而 `useBlocker` 在 react-router v7 內部透過 `useEffect` 延遲註冊新的 shouldBlock。`setSaved(true)` 後緊接 `navigate(-1)` 時，router 內部 blocker 仍是舊的「!saved=true（攔截）」狀態 → 誤跳警告。

## 修復

### Fix 1：明確驗證提示

```js
if (!form.title?.trim()) { alert('請輸入標題'); return }
if (!form.date) { alert('請選擇開始日期'); return }
```

### Fix 2：useLeaveGuard 改用 ref

```js
const savedRef = useRef(false)
const shouldBlock = useCallback(() => !savedRef.current, [])
const blocker = useBlocker(shouldBlock)
// ...
return [blocker, useCallback(() => { savedRef.current = true }, [])]
```

- `shouldBlock` 用 stable `useCallback` 只註冊一次
- `savedRef.current` 同步讀寫，不受 render lifecycle 影響
- 每次 router 呼叫 shouldBlock 都能讀到最新值
