## 變更摘要

重構 `.claude/rules/` 工作流規則：

- **刪除** `core.md`（OpenSpec 工作流）
- **刪除** `global.md`（Git 分支規則）
- **新增** `workflow.md`（整合兩者）
- **更新** `CLAUDE.md` 工作流摘要

## 重構動機

原 `core.md` 與 `global.md` 內容大量重疊（嚴禁行為、推送規則），且兩套工作流（OpenSpec + git 分支）完全分離，導致：

1. 重複提醒，使用者感受到噪音
2. 「新增功能」未與「開 change」連動
3. 修改/執行計畫前無分支確認機制

## workflow.md 新增重點

1. **統一關鍵字觸發表**：git 流程 + OpenSpec 流程同一張表
2. **新增功能 = 切分支 + 開 change**：兩步合一
3. **分支安全閘門**：修改計畫、執行計畫前必須 `git branch --show-current`，在 `main` 或 `dev` 強制停止
4. **嚴禁行為合併**：不再分散在兩個檔案
5. push/README 規則移除（指向 deploy.md）

## 影響範圍

- 僅影響 AI 行為規則，不影響任何程式碼或部署
