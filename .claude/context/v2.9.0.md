# Context: v2.9.0 — 桌機版面置中

## 背景

使用者在桌機瀏覽器（橫式寬螢幕）開啟應用時，內容拉滿整個 viewport 寬度，視覺體驗差。目標：在桌機上以「手機直式欄框居中」的方式呈現，欄框寬度依視窗高度從標準手機比例反推，左右兩側留白，手機上則完全零回歸。

## 改動檔案

- **`frontend/src/main.jsx`**：新增 `pickColWidth()`，React render 前同步設定 `--col-max-w` / `--col-half-w` CSS 變數。依寬高比判斷手機/桌機，避免日後 resize 或換頁造成欄寬跳動。
- **`frontend/src/components/Layout.jsx`**：置中欄加 `width: '100%'`，修正 flex item 在 `#root { display:flex; flex-direction:column }` 下需顯式設寬才填滿 `maxWidth` 的 bug。
- **`frontend/src/components/FabAction.jsx`**：`right` 從 `16px` hardcode 改為 `max(16px, calc(50vw - var(--col-half-w) + 16px))`，對齊欄右緣。
- **`frontend/src/components/SidebarNav.jsx`**：漢堡按鈕 `left` 從 `16` hardcode 改為同公式，對齊欄左緣（原本在桌機欄框之外）。
- **`frontend/src/pages/Management.jsx`**：pill tab 從 inline px（fontSize:12 / padding:'6px 4px'）改 Tailwind rem（text-xs py-1.5 px-1），配合移除 font-size 縮放後確保一致。
- **`openspec/changes/17-桌機版面置中/`**：spec / design / tasks（change 開立、方案取捨記錄、task 全完成）。

## 關鍵設計決策

**決策 1：欄寬從視窗「高度」反推，不用寬度**
桌機橫式螢幕寬度遠大於手機，用寬度判斷會誤判。用高度反推「這個視窗高度下，標準手機比例對應的寬度是多少」，選最接近 430px 的比例。手機因為寬高比 ≤ 9/16，直接用全寬。

**決策 2：main.jsx 同步執行、CSS 變數、整個 session 固定**
若在 React render 之後設定欄寬，會有一幀「閃爍」（先全寬再縮）。同步設到 `<html>` style，所有頁面共用，整個 session 不重算，避免 resize 時欄寬跳動影響排版。

**決策 3：SidebarNav 漢堡按鈕也需要跟隨公式**
原設計文件寫「漢堡鈕維持 left:16 視窗角落」，實際上桌機欄框可能從 viewport 中央 ~740px 開始，left:16 的漢堡完全在欄框外。與使用者確認後改為跟隨欄左緣。

**決策 4：不做 font-size 縮放**
曾嘗試 `html { font-size: scale * 16 }` 讓 rem 等比縮放，但 inline px 值不會跟著縮，Tailwind rem 與 inline px 混用時視覺不一致。移除縮放，改確保元件統一用 rem（Tailwind class）或統一用 px。

## 學習日誌

### 問題 1：Layout 欄位不填滿 maxWidth

**遇到的問題**：桌機上欄框看起來比設定的 CSS 變數寬度窄，內容沒有填滿。

**試過的方法**：用 DevTools 檢查 `--col-max-w` 值是否正確（值正確）、確認 Layout.jsx 的 `maxWidth` 是否讀到（有讀到）。

**最終解法**：在 Layout.jsx 加 `width: '100%'`。

**根本原因**：`#root { display:flex; flex-direction:column }` 讓 Layout 的 column div 成為 flex item。在 flex column 容器裡，flex item 有 `margin-inline: auto` 時，寬度退回內容寬（min-content），而不是撐到 `maxWidth`。加上 `width: 100%` 後，flex item 先想要填滿容器，再被 `maxWidth` 限制到欄框寬度，`marginInline: auto` 才能正確置中。

### 問題 2：漢堡按鈕在桌機跑到欄框外

**遇到的問題**：桌機上欄框置中後，漢堡按鈕（left:16）仍在 viewport 最左邊，與欄框完全脫節。

**試過的方法**：先確認 FAB 的 `right` 公式是否有效（有效）。

**最終解法**：漢堡按鈕套同公式：`left: max(16px, calc(50vw - var(--col-half-w) + 16px))`。

**根本原因**：`position:fixed` 的元件以 viewport 為基準，不受置中欄影響，必須各自用公式跟隨欄位邊緣。

### 問題 3：手機門檻從 520px 改為寬高比

**遇到的問題**：大螢幕手機（如螢幕寬 > 520px 的 tablet）被誤判為桌機，套了欄框置中。

**最終解法**：改用 `W / H <= Math.max(...PHONE_RATIOS)` 判斷：比例 ≤ 9/16 的直立螢幕（手機/直立 tablet）直接全寬，其他走桌機欄框邏輯。

**根本原因**：手機的本質特徵是直立比例，不是絕對寬度。
