## 核心問題與根本解法

### 問題

`googleapis` / `gaxios@6+` 在 Zeabur Node.js 18 環境全面 Premature close：

1. **token 端點**：`google-auth-library@9.x` `JWT.refreshTokenNoCache()` 硬編碼廢棄端點 `www.googleapis.com/oauth2/v4/token`（aud 欄位也同），新端點驗 aud 不符直接關閉連線
2. **Calendar API 呼叫**：`gaxios@6+` 切換到 `globalThis.fetch`（undici），undici 對 Zeabur NAT TCP 半關閉敏感

### 解法

完全不走 `google-auth-library` + `gaxios`，全改用 Node.js 原生 `crypto` + `https.request`：

- `server/config/googleAuth.js`：自簽 JWT assertion（aud = `oauth2.googleapis.com/token`）+ `https.request` 換 token；新增 `getToken()` 和 `calendarApiRequest()` helpers
- `server/services/calendarService.js`：全面改寫，所有 Calendar API 呼叫改用 `calendarApiRequest`

詳細說明：`.claude/context/googleapis-gaxios-zeabur.md`
