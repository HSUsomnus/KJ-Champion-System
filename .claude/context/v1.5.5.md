# v1.5.5 上下文

日期：2026-03-22
分支：staging

## 變更摘要

維護性清理：移除 Google Sheets 死碼，修正環境變數命名語意錯誤。

## 詳細變更

### 刪除
- `server/services/sheetService.js`：成員邏輯已全面改用 `memberDbService.js`（Supabase DB），sheetService 無任何 require，屬死碼

### 修改 `server/config/googleAuth.js`
- 移除 `getSheetsClient()`（死碼，僅被 sheetService 呼叫）
- 移除 `getSheetConfig()`（死碼，且內部 throw MEMBER_SHEET_ID 未設定的錯誤）
- 移除 Service Account 兩個 auth path 中的 `spreadsheets` scope（Sheets API 已不再使用）
- 更新 `module.exports`（移除 getSheetsClient、getSheetConfig）

### 修改 `server/routes/financial.js`
- `/api/financial/sheet-view-url` fallback：`MEMBER_SHEET_ID` → `FINANCIAL_SHEET_ID`
- 語意修正：MEMBER_SHEET_ID 是成員資料表，不是財務試算表

### 修改 `.env`
- `MEMBER_SHEET_ID` → `FINANCIAL_SHEET_ID`
- 移除 `MEMBER_SHEET_NAME`（僅被 sheetService 使用，已刪除）
- 更新註解說明用途

### 修改 OpenSpec tasks.md
- 補記 2e.9（計畫外維護）：詳細記錄以上清理動作

## 影響評估

- 無任何現有功能破壞：sheetService 本來就無 require
- Drive API 上傳財務試算表不受影響（走 getDriveClient，非 getSheetsClient）
- `.env` 的 `FINANCIAL_SHEET_ID` 值與原 `MEMBER_SHEET_ID` 相同，功能不變
- 需重啟本機伺服器（googleAuth.js 有改動）
