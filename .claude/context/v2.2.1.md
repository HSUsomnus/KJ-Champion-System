# v2.2.1 — EventDetail FAB 加紅色刪除按鈕

**Tag**: v2.2.1
**Date**: 2026-04-25
**Type**: hotfix（main UX 缺漏修補）
**Branch flow**: main → hotfix/event-detail-delete-fab → main

## 背景

行程詳情頁（`/event/:id`）右下 FAB 在 v2.2.0 之前長期只有「編輯」與「分享」兩個操作，缺少刪除入口。後端 `DELETE /api/calendar/events/:eventId` 與前端 `api.deleteEvent(eventId)` 早已存在（calendar.js / api.js 既有實作），這純粹是前端 FAB UX 缺漏 — 使用者要刪行程必須繞道行事曆其他流程，或被迫只能編輯不能刪。

## 修法

`frontend/src/pages/EventDetail.jsx`：

1. **新增 `handleDelete` 處理函式**：
   - `window.confirm` 二次確認（含行程標題提示，文案：`確定要刪除「{title}」？此操作無法復原。`）
   - 呼叫 `api.deleteEvent(event.id)`
   - 成功 → `navigate('/calendar')`
   - 失敗 → `alert(res.error || res.message || '刪除失敗')` 直接顯示後端錯誤
   - 例外 → `alert(err.message || '刪除失敗')`
   - 不 fallback 成其他行為（依 workflow.md 錯誤處理規則）

2. **`fabItems` 第三項加「刪除」**（生日事件不加）：
   - `label: '刪除'`
   - `labelColor: '#dc2626'`（label 文字紅）
   - `labelBorderColor: '#dc2626'`（label 邊框紅）
   - `btnColor: '#dc2626'`（icon 紅）
   - `btnBorderColor: '#dc2626'`（按鈕邊框紅）
   - `btnBg`：保留預設白底（避免大紅塊太刺眼，與「編輯/分享」白底一致，僅靠紅色 icon + 紅邊 + 紅文字標示警示）
   - `icon`：垃圾桶 SVG（trash-2 樣式）

3. **生日事件 (`isBirthdayEvent`)**：因為生日是動態從成員生日生成，不存在於 events 表，無法刪除，所以 `isBirthdayEvent` 分支只保留「分享」一項，不顯示刪除按鈕。

## 驗證

- ⬜ dev / 本機 server：清 SW 後進入任一行程詳情頁，右下 FAB 展開應出現第三顆紅色「刪除」按鈕
- ⬜ 點刪除 → 跳 confirm dialog → 取消：留在原頁
- ⬜ 點刪除 → confirm 確認 → 刪除成功 → 自動跳 `/calendar` → 行程從月曆消失
- ⬜ 後端關掉模擬刪除失敗 → 顯示錯誤訊息不 redirect
- ⬜ 進入生日事件詳情頁 → FAB 只有「分享」一項，無刪除按鈕
