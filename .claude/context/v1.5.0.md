## 主要變動

### 1. ngrok 整合

**新增檔案**：`ngrok.yml`
- 定義 ngrok 通道設定（port 8080）
- 支援從環境變數 `NGROK_AUTHTOKEN` 讀取 token
- 附有詳細使用說明注釋

**`package.json` 變動**：
- 移除廢棄的 `build` 與 `postinstall` 腳本（原本指向已廢棄的 React frontend）
- 新增 `"ngrok": "ngrok http 8080"` 腳本
- 新增 `"dev:ngrok": "concurrently \"npm run dev\" \"npm run ngrok\""` 腳本
- 新增 `concurrently@^8.2.2` 為 devDependency

**使用方式**：
```bash
npm run dev:ngrok   # 同時啟動伺服器 + ngrok（推薦）
npm run ngrok       # 只啟動 ngrok
```

## 架構現況（v1.5.0）

- **前端**：純 HTML/JS/CSS（`public/`），React 版已廢棄
- **後端**：Node.js + Express.js（`server/`）
- **身份驗證**：LINE Login OAuth，不依賴 LIFF SDK
- **資料庫**：Supabase (PostgreSQL) + Google Calendar + Google Sheets
- **部署**：Vercel（前後端統一）
- **本機開發**：ngrok（內網穿透）

---

## ngrok 使用情境

| 情境 | 說明 |
|------|------|
| 測試 LINE Login | 需要 HTTPS 回調 URL，ngrok 提供公開 HTTPS |
| 測試 LINE BOT Webhook | LINE 平台需要公開 HTTPS URL 發送事件 |
| 手機 LINE App 測試 | 手機無法連 localhost，需要 ngrok URL |

---

## 注意事項

- 免費版 ngrok 每次重啟都會換 URL，需重新更新 LINE Developers Console
- 測試完畢後立即關閉 ngrok，不要長時間開著
- `NGROK_AUTHTOKEN` 應加入 `.env`，不得推上 GitHub
