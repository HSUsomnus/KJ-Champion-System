## 做了什麼

### PWA 支援
- 新增 `public/manifest.json`：應用名稱「康九冠軍夥伴系統」、短名「康九冠軍系統」、LINE 綠主題色、獨立顯示模式
- 新增 `public/sw.js`：Service Worker，Cache-first 策略，靜態資源預快取（34 個檔案），CACHE_NAME = kj-cache-v4
- 所有 14 個 HTML 頁面新增 PWA meta 標籤（manifest、theme-color、apple-mobile-web-app 系列）
- `server/server.js`：新增 `/sw.js` 的 no-cache header，確保瀏覽器永遠取得最新 SW
- PWA 可安裝至桌面，已確認 Android/iOS/Chrome 電腦皆可安裝
- 桌面安裝後自動縮成手機比例（390x844，`window.resizeTo()`）

### liff.js 改名為 auth.js
- `public/js/liff.js` → `public/js/auth.js`
- 內部變數重命名：`liffInitialized→authInitialized`、`liffUserId→currentUserId`、`liffProfile→currentProfile`、`initLIFF()→initAuth()`
- **保留** `window.LIFF` 和 `window.liff` 對外介面（110+ 處引用，不動）
- 9 個 HTML 檔案的 script src 改為 `/js/auth.js`
- SW STATIC_ASSETS 中 `/js/liff.js` → `/js/auth.js`

### 登入 overlay 修正（重要）
- **問題**：index.html 在未登入時一直轉圈圈，不顯示登入畫面
- **根本原因 1**：`initAuth()` 在正式模式下同步執行，`liffReady` 事件在 `entryGate` 監聽器註冊前就觸發，事件被遺漏
- **修正 1**：用 `setTimeout(0)` 包住 `liffReady` dispatch（auth.js 第 151-155 行）
- **根本原因 2**：`entry-overlay` HTML 元素預設帶 `entry-overlay-hidden` class（隱藏），即使事件遺失也不會顯示登入畫面
- **修正 2**：`index.html` 的 `entry-overlay` 移除預設的 `entry-overlay-hidden` class，改為「預設顯示，確認登入後才隱藏」（Fail-safe 設計）

### 其他
- `package.json`：name 改為 `kj-champion-partner`，description 更新，移除 liff 關鍵字
- `README.md`：完全重寫，反映目前技術架構（無 LIFF SDK）

## 為什麼改

- PWA：使用者要求可安裝至手機桌面，提升使用體驗
- liff.js 改名：專案已不使用 LINE LIFF SDK，改名讓程式碼更能反映實際用途
- 登入 overlay 修正：主頁未登入時轉圈圈是 bug，使用者無法知道要先登入

## 踩過的坑

1. **Manifest icon 尺寸宣告錯誤**：最初宣告 192x192/512x512，但 logo.png 實際為 1024x1024，Chrome 顯示警告 → 修正為實際尺寸
2. **`purpose: "any maskable"` 問題**：Chrome 不建議合併，改為兩個獨立 icon 項目（一個 `any`，一個 `maskable`）
3. **window.resizeTo() 限制**：Chrome 安全機制限制 PWA 視窗大小調整，最終在 SW 快取更新後生效
4. **SW 快取阻止修正生效**：修改 auth.js 後，SW 仍服務舊版快取 → 升級 CACHE_NAME v3→v4 強制更新
5. **liffReady 事件時序問題**：async function 中沒有 await 的路徑實際同步執行，setTimeout(0) 確保正確時序
6. **測試環境混淆**：使用者一直在 Vercel 正式環境（line-liff-calendar.vercel.app）測試，誤以為是本機 → 解釋兩個環境的差異

## 未來維護注意

- SW 快取版本：每次修改靜態資源後，需升級 `public/sw.js` 的 `CACHE_NAME`（目前 kj-cache-v4）
- `window.LIFF` 介面不能改（110+ 外部引用），只能在 auth.js 內部調整
- `entry-overlay` 現在預設顯示（已移除 `entry-overlay-hidden`），`showMain()` 負責隱藏它
- `invite-share.html` 仍使用 LINE 官方 LIFF SDK（不能改）
- 本機測試 LINE OAuth 需要 `APP_URL` 指向 localhost，且要在 LINE Developer Console 新增 localhost 為 Callback URL
