## 背景

v2.2.0 已上線後端：`node-cron` 排程、`system_settings` 表、3 個 API（GET/PUT settings + POST 手動推播）。後端在 dev 上連續多日 23:30 推播驗證通過，prod 也已驗證手動推播 + 字卡內容（dev seed event 觸發）。

1. **開發者設定頁** `/agenda-settings`（toggle / 時間 / 對象 / 立即推播 / Eruda toggle）
2. **FabNav 開發者入口**（`role === '開發者'` 才顯示）

4. **Eruda inline loader**（index.html 內，`?eruda=1` URL 或 `localStorage.erudaEnabled='true'` 任一觸發載入 cdn.jsdelivr.net/npm/eruda）
5. **PWA `mobile-web-app-capable` meta tag**（補 `apple-mobile-web-app-capable` deprecated 警告，新舊並存）

## 關鍵設計決策

### 1. AgendaSettings 同時整合「Eruda toggle」與「推播設定」

兩者都是「僅開發者可用」的功能，與其分兩頁，合併為單一「開發者設定」頁。減少 FabNav 入口數量、降低 onboarding 中其他角色看到的視覺噪音。

### 2. Toggle 用 inline style 不用 Tailwind class

Tailwind JIT 在動態 class 名（如 `translate-x-${on ? '5' : '0'}`）會漏掃，導致 production build 沒有對應 class，knob 不會動。改用 `style={{ transform: ... }}` 直接寫值，繞過 JIT 限制。（commit `9c29bc7` 修法）

### 3. Eruda 載入策略

「URL 觸發」用於急救（手機現場 debug 不用先去設定頁開）；「localStorage 觸發」用於常駐（一次設定後每次開都載）。兩條件 OR 滿足即載入。CDN `cdn.jsdelivr.net/npm/eruda`，無 fallback（debug 工具，CDN 掛掉影響低）。

### 4. PWA meta tag 雙寫

`apple-mobile-web-app-capable` 在最新 Chrome DevTools 列為 deprecated（建議改 `mobile-web-app-capable`）。但 iOS Safari 仍認舊 meta。所以兩個並列，新舊瀏覽器都吃。

### 5. 「立即推播」直接 POST，無 confirm dialog

依使用者明確需求：應與 DevTools 手動 fetch 行為一致 — 跳過時間設定，直接用 UI 目前選的對象推播。不加 confirm 是設計選擇（測試流暢度優先，誤觸風險使用者自己控制）。

### 6. 不覆寫既有 onboarding 資料的 seed 腳本

`scripts/seed-dev-agenda-test.js` 用 `INSERT ... ON CONFLICT (line_id) DO UPDATE SET role = '開發者'`（只更新 role，不覆寫姓名/email/phone/birthday/courseRecord）。已 onboarding 過的 dev DB 帳號重跑 seed 不會被假資料蓋掉。

## dev 驗證流程

dev DB 物理隔離（v2.1.0）→ dev DB 為空。要測 developer-only 頁面必須：

1. PC 跑 `node scripts/seed-dev-agenda-test.js U<lineUserId>`（或 psql 直執行 SQL）
2. dev 站清 SW + 強制重整 → LINE 重登入
3. FabNav 出現「開發者設定」齒輪入口
4. 進設定頁讀寫 + 立即推播（會推給 developer 自己）→ 收到 4/27 Flex 字卡
5. Eruda toggle 開 → 重整 → 右下綠按鈕
6. localStorage swap lineUserId 為 `U77777777777777777777777777777777`（一般人測試帳號）→ 直訪 `/agenda-settings` → 應顯示「無存取權限」

全部驗證通過（2026-04-26 PC session）。
