# v1.5.4 上下文

## 版本資訊
- **版本**：v1.5.4
- **日期**：2026-03-22
- **分支**：main + staging

## 本版變更

### 移除雙寫服務（dualWriteService）

**原因**：雙寫驗證（tasks 2e.6、2e.7）遲遲無法完成，決定跳過直接移除，簡化架構。

### 刪除的檔案
- `server/services/dualWriteService.js`

### 修改的檔案

**`server/services/eventDbService.js`**：
- 移除 `require('./dualWriteService')`
- `upsertEvents()`：`dualWrite()` → 直接 `_upsertEventsToPool(db.getClient, events)`
- `deleteEventsNotIn()`：`dualWrite()` → 直接 `runDelete(db.query.bind(db))`
- `deleteEventById()`：`dualWrite()` → 直接 `db.query()`

**`server/services/memberDbService.js`**：
- 移除 `require('./dualWriteService')`
- `createMember()`：`dualWrite()` → 直接 `db.query()`
- `updateMember()`：`dualWrite()` → 直接 `db.query()`
- `updateMemberRole()`：`dualWrite()` → 直接 `db.query()`
- `updateFinancialAmount()`：`dualWrite()` → 直接 `db.query()`

## 目前資料庫架構
- **唯一資料庫**：Zeabur PostgreSQL（`DATABASE_URL`）
- Supabase 不再接收任何寫入

## 注意事項
- Vercel 環境變數 `DUAL_WRITE_ENABLED` 和 `SUPABASE_BACKUP_URL` 已無作用，可從 Vercel 控制台移除（非必要）
