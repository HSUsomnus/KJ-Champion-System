# v2.10.0 — Change 19「主頁快捷資訊」

## 背景

使用者希望主頁一眼看到個人財力狀態，並快速跳至財力上傳頁；同時從主頁直達三個常用系統（LINE Bot、Google 行事曆、安裝 PWA），減少進入各頁面的層數。

- `frontend/src/pages/Home.jsx` — 全面改寫：歡迎卡整合財力金額、系統連結區（3 個圖示方塊）、今日行程移至下方。inline SVG 取代 emoji（UIDESIGN 規範）。PWA 按鈕依狀態 dialog 提示。

## 關鍵設計決策

**beforeinstallprompt 全域捕捉**：瀏覽器在頁面載入早期就觸發此 event，React useEffect 在 DOM mount 後才執行，必然 miss。解法：在 `main.jsx` createRoot 前加 listener 存入 `window.__pwaInstallPrompt`，組件 mount 時直接讀取，再同時掛 future 事件 listener 防止重複觸發的場合。

**PWA dialog 而非 disabled**：原設計 standalone 模式 disabled 按鈕，但無法告知使用者原因。改成統一可點擊，依狀態（已安裝 / 不支援）彈出 UIDESIGN ConfirmDialog 規格的 dialog，比 silent disabled 更友善。

**手機 / PC 不同訊息**：`window.matchMedia('(pointer: coarse)')` 偵測觸控裝置，手機顯示「請使用其他瀏覽器」，PC 明確指定「Chrome 或 Edge」。

**系統連結 URL 走後端 API**：LINE Bot URL 與 Google Calendar ID 存在後端 env var，前端無法直接取得，需透過 `GET /api/line/system-links` 橋接。endpoint 無需驗證（URL 本身無敏感資訊）。

## 學習日誌

### 問題 1：emoji 被 UIDESIGN 禁用
**遇到的問題**：第一版用 emoji（💬 📅 📲）做 icon，使用者反映風格不對。
**試過的方法**：AskUserQuestion ASCII mockup → 使用者要求「畫出來，不是文字」→ 建 HTML prototype。
**最終解法**：UIDESIGN.md 明確規定「禁止使用 emoji 作為 icon，全部使用 inline SVG」，換成 stroke 線條 SVG，顏色 `#4A7C59`（accent）。
**根本原因**：實作前未充分讀 UIDESIGN.md 的禁止事項段落。

### 問題 2：PWA install prompt 無法觸發
**遇到的問題**：手機與電腦點擊安裝按鈕都沒有跳出 install prompt。
**試過的方法**：useEffect 裡加 `beforeinstallprompt` listener。
**最終解法**：`main.jsx` 在 React render 前預攔截，存入 `window.__pwaInstallPrompt`，組件 mount 時讀取。
**根本原因**：`beforeinstallprompt` 在瀏覽器決定 PWA 安裝條件滿足時立即觸發（頁面載入早期），比 React component mount 早很多，useEffect 必然 miss。
