# v1.1.0 上下文記憶

git tag: v1.1.0
日期：2025
相關 commits：e7e8a53

---

## 做了什麼

- **完全移除 LINE LIFF SDK**，改用自製 LINE Login OAuth + LINE URL Scheme
- 新增 `public/js/liff.js`（後來改名為 auth.js）：
  - LINE Login OAuth 流程（導向 `/api/auth/line-login` → LINE 授權 → 回調帶 userId）
  - 提供 `window.LIFF` 對外介面（讓所有頁面無需修改）
  - 提供 `window.liff` 相容層（讓舊的 `liff.xxx()` 呼叫繼續運作）
  - 開發模式支援（`?dev=1` 或 `APP_RUN_MODE='development'`）
- 新增 `server/routes/auth.js`：LINE OAuth 2.0 回調處理
- 分享功能改用 LINE URL Scheme（`https://line.me/R/share?text=...`），手機自動開啟 LINE 分享畫面
- 電腦分享：fallback 到 Web Share API 或複製到剪貼簿

---

## 為什麼改

- LINE LIFF SDK 限制多（必須在 LINE App 內開啟才能使用 LIFF 功能）
- 改用 LINE Login OAuth 後，所有頁面都可在一般瀏覽器使用
- LINE URL Scheme 分享不需要 LIFF，手機和電腦都能分享

---

## 踩過的坑

1. **`window.LIFF` 有 110+ 處引用**：不能改對外介面，必須保留相同的 API 名稱
2. **LINE URL Scheme 無法傳 Flex Message**：只能傳純文字，Flex Message 需要 LIFF shareTargetPicker（v1.3.0 的邀請字卡功能）
3. **開發模式需求**：本機開發沒有 LINE 帳號時需要模擬登入 → 加入 `?dev=1` 模式

---

## 未來維護注意

- `window.LIFF` 和 `window.liff` 的對外介面**絕對不能改**（會導致所有 JS 檔案出錯）
- LINE OAuth 的 redirect_uri 必須在 LINE Developer Console 的 Callback URL 中登記
- `APP_URL` 環境變數決定 OAuth callback 的目標域（本機測試需改為 localhost）
- `invite-share.html` 是唯一仍使用真正 LIFF SDK 的頁面（不在 v1.1.0 移除範圍內）
