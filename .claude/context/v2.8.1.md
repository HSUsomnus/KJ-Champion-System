# v2.8.1 上下文

## 背景

v2.8.0 部署到 Zeabur 後 15 分鐘內「部署失敗」。v2.7.0（5 小時前部署）正常運行，v2.8.0 build 直接失敗。初步懷疑是 change 17 程式碼問題，但檢查後 server.js / package.json / zbpack.json 均無變動，後端程式碼邏輯正常。

## 改動檔案

- `package.json` — engines 加入 `"npm": "10"`，指定 npm 版本，繞過 Zeabur 基礎映像 bug

## 關鍵設計決策

不退版（git reset --hard 206331e），而是修根本原因。這樣保留了 v2.8.0 全部功能（export-backup-csv、SidebarNav 等），不需重新 merge。

## 學習日誌

### 問題 1：v2.8.0 Zeabur 部署失敗，v2.7.0 正常

**遇到的問題**：Zeabur dashboard 顯示「部署失敗」，日誌頁面顯示「等待建置日誌中... 找不到日誌」，程式碼分析找不到任何可能導致 build 失敗的變更。

**試過的方法**：
- 比對 v2.7.0 vs v2.8.0 所有 server/ 檔案差異 → 程式碼看起來正常
- 確認 package.json / package-lock.json / zbpack.json 無變動
- 查 Zeabur 官方文件的 build 流程

**最終解法**：查 Zeabur 論壇，找到已知 bug：基礎 Docker image 更新後，全域 npm 安裝缺少 `promise-retry`（`@npmcli/arborist` 依賴），導致 `npm update -g npm` 執行失敗。

Fix：在 `package.json engines` 加 `"npm": "10"` → Zeabur 改執行 `npm install -f -g npm@10` 繞過壞路徑。

**根本原因**：Zeabur 基礎映像問題，與我們的程式碼完全無關。v2.7.0 在映像出問題前部署所以正常；v2.8.0 在映像出問題後部署所以失敗。dev 之前也崩潰是同一原因。

## 驗證結果

推送後等待 Zeabur 自動重新部署，確認 `kj-champion-system.zeabur.app` 後端正常回應。

## 後續注意

- 此 npm 版本鎖定（`"npm": "10"`）可能需要在未來 Zeabur 修復基礎映像後評估是否移除（低優先，保留無害）
- 若未來 Zeabur 再次出現 build 失敗，先查論壇是否有新的映像問題
