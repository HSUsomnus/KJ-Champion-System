# v2.7.0 上下文 — change 16「備份 DB 同步」

## 背景

原始需求：prod DB 寫入時同步備份到第二個 PostgreSQL，讓 dev 環境能從備份 DB 拉取真實資料測試。
初版方案是 write-through queue（在 db.js 攔截所有 INSERT/UPDATE/DELETE，異步寫到備份 DB），但發現只能抓新寫入，既有的 members/financial_documents 幾千筆資料永遠無法備份進來。因此在實作途中改架構為「定時全量覆蓋」：每 8 小時從 prod DB 全量讀取 → TRUNCATE 備份 DB → INSERT 全部資料。

## 關鍵設計決策

**為何捨棄 write-through queue**：
- Queue 只抓「系統啟動後的新寫入」，既有資料（members、financial_documents）完全無法備份
- events 有備份是因為 calendarSync 每分鐘觸發 upsert，恰好在 queue 存在的那段時間就有寫入
- 全量覆蓋的缺點是 8 小時會有舊資料的 window，但對這個系統規模（數十到數百筆）完全可接受

**Zeabur 新架構（3 個 PostgreSQL 服務在 kj-champion 專案）**：
- `postgresql`（prod DB，內網 only）
- `postgresql-backup`（備份 DB，內網 only）
- `postgresql-dev`（dev DB，**公網開啟**，供 dev backend 跨專案連線）

**Zeabur 環境變數注意**：`${postgresql-backup.POSTGRES_CONNECTION_STRING}` 跨服務引用無法解析，必須手動填入實際連線字串。

**Admin API 設計**：
- `POST /api/admin/sync-prod-to-backup`：手動觸發全量備份（不等排程）
- `POST /api/admin/sync-backup-to-dev`：把備份 DB 複製到 dev DB（需 DEV_DATABASE_URL）
- `GET /api/admin/backup-status`：查詢備份 DB 各 table 筆數
- 全部由 `Authorization: Bearer <ADMIN_SECRET>` 保護

## 學習日誌

### 問題 1：PowerShell inline env var 語法錯誤
**遇到的問題**：`TARGET_DB_URL="..." node scripts/init-db.js` 在 PowerShell 報 parse error
**試過的方法**：各種引號組合
**最終解法**：`$env:TARGET_DB_URL="..."; node scripts/init-db.js`
**根本原因**：PowerShell 不支援 Unix 的 `VAR=value command` 語法

### 問題 2：`Cannot find module 'pg'`
**遇到的問題**：在根目錄執行 `node scripts/init-db.js` 報找不到 pg
**試過的方法**：直接執行
**最終解法**：先 `npm install` 安裝根目錄的 pg 依賴（scripts/ 的 Node 腳本需要從根目錄讀 package.json）
**根本原因**：`scripts/` 沒有自己的 node_modules，依賴根目錄 package.json

### 問題 3：Zeabur BACKUP_DATABASE_URL 沒有解析
**遇到的問題**：在 `kj-champion-system` 服務設了 `BACKUP_DATABASE_URL=${postgresql-backup.POSTGRES_CONNECTION_STRING}` 但後端日誌顯示「BACKUP_DATABASE_URL 未設定」
**試過的方法**：Zeabur 跨服務變數引用
**最終解法**：直接填入完整連線字串 `postgresql://root:<password>@postgresql-backup.zeabur.internal:5432/zeabur`
**根本原因**：Zeabur 的「已暴露」（Exposed）是指同專案其他服務可以引用這個變數，但手動設定的環境變數欄位不會自動解析 `${}` 語法，必須填實際值

### 問題 4：Admin API 401 未授權（PowerShell 變數插值）
**遇到的問題**：`$secret = "rB4!xK9$mPq2..."` 設定後 curl 回傳 401
**試過的方法**：雙引號字串
**最終解法**：`$secret = 'rB4!xK9$mPq2...'` 改用單引號
**根本原因**：PowerShell 在雙引號字串中把 `$m` 解讀為變數展開，導致 ADMIN_SECRET 被截斷

### 問題 5：備份 DB 沒有 schema（Zeabur web viewer 顯示「未找到資料庫」）
**遇到的問題**：備份 DB 建立後不知道怎麼初始化 schema（沒有公網、沒有 pg_dump）
**試過的方法**：試圖用 Zeabur 的 DB 管理介面
**最終解法**：臨時開公網 → `scripts/init-db.js` 套用所有 migration → 關公網
**根本原因**：Zeabur web DB viewer 需要連線埠轉送（公網），不能直接存取內網 DB
