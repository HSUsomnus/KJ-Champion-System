# v1.5.6 上下文

日期：2026-03-22
分支：staging

## 變更摘要

維護性清理：精簡 `.env` 死碼、補齊缺漏環境變數；新增 Claude Code hooks 強化 OpenSpec 工作流程守護。

## 詳細變更

### 修改 `.env`

- 移除 `ADMIN_LINE_USER_IDS`（死碼，程式碼中無任何地方讀取）
- 移除 `DUAL_WRITE_ENABLED`、`SUPABASE_BACKUP_URL`（雙寫服務已於 v1.5.4 從程式碼完全移除）
- 移除舊版 Google API 分拆變數：`GOOGLE_SERVICE_ACCOUNT_EMAIL`、`GOOGLE_PRIVATE_KEY`、`GOOGLE_PROJECT_ID`（方案 2，現行程式碼只讀取 `GOOGLE_SERVICE_ACCOUNT_JSON`）
- 新增 `FRONTEND_URL`（CORS middleware 白名單用）
- 新增 `CRON_SECRET`（目前留空；未設定時 Cron endpoint 無保護）

### 新增 `.claude/hooks/`

- `openspec-code-guard.js`：PreToolUse hook，修改 `server/` 或 `public/` 程式碼前自動注入 OpenSpec 授權確認提示
- `openspec-sync-reminder.js`：PostToolUse hook，偵測 openspec 文件被修改後提醒執行 `/openspec-sync-docs`

### 更新 OpenSpec 文件

- `tasks.md`：補記 2e.10（計畫外維護）— 記錄 `.env` 清理的 5 項已完成任務
- `design.md`：更新 Context 進度說明，反映 `.env` 清理完成
- `specs/zeabur-backend-deploy/spec.md`：env.example 說明移除 Supabase，改為 Zeabur PostgreSQL

## 影響評估

- `.env` 清理不影響任何現有功能（移除的變數均為死碼或已廢棄服務的設定）
- `CRON_SECRET` 留空：Cron endpoint 目前無認證保護，部署 Zeabur staging 時應填入隨機字串
- Claude Code hooks 不影響程式碼本身，只改變 AI 輔助開發的工作流程守護
