# v1.5.3 上下文

## 版本資訊
- **版本**：v1.5.3
- **日期**：2026-03-22
- **分支**：staging

## 本版變更

### 問題根因
`server/services/dualWriteService.js` 在 staging 分支中遺失。
commit `bf9db45`（「staging 雙寫整合」）修改了 `eventDbService.js` 和 `memberDbService.js` 加入了 `require('./dualWriteService')`，但未將 `dualWriteService.js` 本身加入 staging 分支。

### 症狀
- 新增、更新、刪除行程時，主庫（Zeabur PostgreSQL）操作正常
- 但 Supabase 備份庫從未收到寫入
- 前端顯示「成功」（因為 calendarService 中的 catch 吞掉了錯誤）

### 修復
從 main 分支複製 `server/services/dualWriteService.js` 至 staging 分支。

### 新增檔案
- `server/services/dualWriteService.js`：雙寫包裝服務
  - 主庫成功後，非同步（fire-and-forget）備份到 Supabase
  - 環境變數：`DUAL_WRITE_ENABLED=true`、`SUPABASE_BACKUP_URL`
  - 匯出：`dualWrite()`、`backupQuery()`、`backupGetClient()`、`isEnabled()`

## 受影響功能
- 新增行程（POST /api/calendar/events）→ 現可同步寫入 Supabase
- 更新行程（PUT /api/calendar/events/:eventId）→ 現可同步
- 刪除行程（DELETE /api/calendar/events/:eventId）→ 現可同步
- 成員相關寫入（memberDbService）→ 現可同步

## 相關檔案
- `server/services/dualWriteService.js`（新增）
- `server/services/eventDbService.js`（引用 dualWriteService）
- `server/services/memberDbService.js`（引用 dualWriteService）
- `server/services/calendarService.js`（呼叫 eventDbService.upsertEvents）
