## 問題描述
使用者登入後 console 顯示「登入成功」，但首頁仍卡在登入 overlay 無法進入主頁。

## 根本原因（Race Condition）
`auth.js` 的 `initAuth()` 用 `setTimeout(0)` 派發 `liffReady` 事件，目的是等 HTML inline script 的 listener 先註冊完。

但在 Vercel CDN 環境下，`<script src>` 標籤（非 async/defer）之間存在網路等待間隙。瀏覽器在等待下一個 JS 檔案下載期間，event loop 會處理 macrotask（包括 `setTimeout(0)`），導致 `liffReady` 在 entryGate listener 還未掛上時就已觸發 → 事件漏接 → overlay 永遠不消失。

## 修復方式
**`public/index.html`**：
- 將 entryGate 的處理邏輯抽成 `handleUser(userId)` 函式
- 在 `liffReady` listener 之後，補查一次 `window.LIFF.isInitialized()`
- 若 auth 已初始化（代表 liffReady 已被漏接），立即呼叫 `handleUser(window.LIFF.getUserId())`

**`public/sw.js`**：
- CACHE_NAME 從 `kj-cache-v5` 升至 `kj-cache-v6`，強制瀏覽器載入新版 index.html
