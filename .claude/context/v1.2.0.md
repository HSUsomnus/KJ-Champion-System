# v1.2.0 上下文記憶

## 做了什麼

- 新增管理中心（`management.html`）：資料頁籤（成員進階資訊）+ 財務頁籤（限 manager/admin）
- 財務上傳系統（`financial-upload.html`、`financial-preview.html`）：Excel 檔案上傳、轉換為 Google Sheet、預覽
- 角色管理 UI：admin 可在管理中心修改成員角色（一般成員/admin/manager）
- 權限系統改用**資料庫角色**（Supabase members 表的 role 欄位），移除原本的環境變數 `ADMIN_LINE_USER_IDS`（僅保留初始 admin 用）
- 新增 manager（管理者）角色，權限介於 admin 和一般成員之間
- 財務金額欄位：上傳頁顯示、管理中心可編輯、財務頁籤卡片顯示
- 分享對話框（`share-dialog.js`）統一各頁面的 alert/confirm UI

---

## 為什麼改

- 管理功能集中到管理中心，避免散落各頁面
- 財務資料需要上傳和預覽功能，原本沒有
- 角色改用資料庫更靈活，不用重新部署就能修改權限

---

## 踩過的坑

1. **multer 安全漏洞**：升級到 multer 2.x 修復
2. **Excel 在手機 LINE 瀏覽器下載問題**：改用直接 URL 導航而非 `window.open`
3. **財務上傳頁在 LIFF 內無法開啟**：需在外部瀏覽器開啟 → 加入導轉邏輯
4. **角色值從英文改為中文**：`admin/manager/member` → `管理員/manager/一般成員`（需確認資料庫中的實際值）

---
