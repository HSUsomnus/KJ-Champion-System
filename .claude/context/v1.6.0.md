## 變更摘要

後端 LINE Login OAuth 新增動態 origin 偵測，callback 後自動 redirect 回發起登入的前端站，解決 DEV/正式站混跳問題。

## 變更檔案

| 檔案 | 變更 |
|------|------|
| `server/routes/auth.js` | 新增 `getClientOrigin()`、`isAllowedOrigin()`、`ALLOWED_ORIGINS` 白名單；state 加入 `frontendOrigin` 欄位 |
| `openspec/changes/07-oauth動態redirect/` | 新增 proposal.md、design.md、tasks.md |
| `openspec/STATUS.md` | 新增 change 07 |

## 技術細節

- 從 HTTP `Origin` / `Referer` header 偵測前端 origin
- 通過 `ALLOWED_ORIGINS` 正規表達式白名單驗證，防止 open redirect
- 白名單包含：`*.pages.dev`、`*kj-champion*`、`localhost`、`127.0.0.1`
- 未通過驗證時 fallback 到 `FRONTEND_URL` 環境變數

## 影響範圍

- 所有使用 LINE Login 的前端站（正式、DEV、本機）都受惠
- 不影響現有功能，純增量改動
- 不需要新增環境變數
