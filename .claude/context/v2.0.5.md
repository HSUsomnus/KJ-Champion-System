# v2.0.5 — Login.jsx 首次登入「建立資料」死循環修復

**Tag**: v2.0.5
**Date**: 2026-04-25
**Type**: hotfix
**Branch flow**: main → hotfix/login-no-profile-redirect-loop → main

## 背景

OpenSpec change 10（Zeabur 專案分離 — dev 與 prod 完全物理隔離）完成 dev DB 隔離後，使用者首次在新 dev 站登入測試時撞到死循環：

1. LINE OAuth 完成 → 回到前端 `/login`
2. AuthContext 呼叫 `getProfile()` → 404（新 dev DB 為空，找不到 member 記錄）
3. Login 頁顯示「首次登入，請建立用戶資料」+「建立資料」按鈕
4. 點「建立資料」→ navigate 到 `/profile/edit`
5. `/profile/edit` 在 `ProtectedRoute` 下 → 看到 `user` 為 null → 踢回 `/login`
6. 進 Login → localStorage 仍有 `lineUserId` → 又走步驟 2 → 再顯示「建立資料」
7. 死循環

## 為什麼 prod 從未撞到

prod DB 永遠有 member 記錄（24 筆真實使用者），LINE 登入回來 `getProfile()` 一定 200 → 一定走 `'has-profile'` 分支 → `login(userData)` 正常呼叫 → user state 有值 → `/profile/edit` 不會被 ProtectedRoute 踢。

OpenSpec change 10 完成 dev 與 prod DB 物理隔離後，新 dev DB 起始為空，這條邊界路徑才**首次被真正走過**，bug 也才暴露。

## 根因

`frontend/src/pages/Login.jsx` 的 `handleConfirm`（v2.0.0 React 前端建立時就在）：

```js
const handleConfirm = () => {
  if (authState === 'has-profile' && userData) {
    login(userData)               // ← 'has-profile' 有呼叫 login
    navigate('/')
  } else {
    navigate('/profile/edit')      // ← 'no-profile' 直接跳轉，沒呼叫 login()
  }
}
```

`/profile/edit` 在 `App.jsx` 的 `ProtectedRoute` 下：

```js
function ProtectedRoute() {
  const { user, loading } = useAuth()
  if (loading) return <載入中...>
  if (!user) return <Navigate to="/login" replace />  // ← user 為 null 就踢
  return <Outlet />
}
```

`'no-profile'` 分支沒呼叫 `login(userData)` → AuthContext 的 user 仍是 null → ProtectedRoute 踢人 → 死循環。

## 修法

`'no-profile'` 分支補一行 `login(userData)`：

```js
} else if (authState === 'no-profile' && userData) {
  // [設計決策] 首次登入也要先 login(userData) 把 user state 設起來
  // 原因：/profile/edit 在 ProtectedRoute 下，user 為 null 會被踢回 /login 形成死循環
  login(userData)
  navigate('/profile/edit')
}
```

`userData` 此時雖然只有 `lineId / displayName / pictureUrl`（沒有完整 member 資料），但已足以讓 `user` 不為 null 通過 ProtectedRoute。`/profile/edit` 頁本身就是設計來處理「首次填表單」流程，POST 成功後 `refreshUser` 會把完整 member 補進來。
