# 修正「開始日期／結束日期／開始時間／結束時間」輸入體驗

> 針對**新增行程**與**行程詳情編輯**中，你框選的這四個欄位：**開始日期 \*、結束日期、開始時間 \*、結束時間 \***，目前設定方式太難用。以下為重新搜尋網路 UX 研究後整理出的**三種修正方案**，擇一實作即可明顯改善。

---

## 一、問題與目標

- **現狀**：四個欄位依序為「開始日期」「結束日期」「開始時間」「結束時間」，點擊後多為滾輪或原生選擇器，在手機上操作費力（滑動難對齊、常用時段無法一鍵選）。
- **目標**：同一塊區域內，讓使用者**少滑動、多點選**，且符合排程情境（常選「某一天」+「常見時段」如 14:00、19:00）。

## 三、方案 1：原生 input + 快捷鍵與 Chip（推薦先做）

**對象**：你框選的「開始日期、結束日期、開始時間、結束時間」四個欄位。

**做法**

1. **日期兩欄**
   - 維持 `<input type="date">`（或改用原生，若目前被自訂滾輪覆蓋），讓裝置自己開日期選擇器。
   - 在「開始日期」同一行右側加兩個小按鈕：**「今天」**、**「明天」**，點一下就把該欄設為今天／明天，減少手動選日。

2. **時間兩欄**
   - 維持 `<input type="time">`，讓裝置自己開時間選擇器。
   - 在「開始時間」「結束時間」**上方或下方**加一排 **Chip**：`09:00`、`10:00`、`14:00`、`19:00`、`20:00`（可依你們常用時段調整）。點某個 Chip 就自動帶入對應欄位；若點「開始時間」的 Chip，就寫入開始時間；若點「結束時間」的 Chip，就寫入結束時間（或設計成：先選開始再選結束，Chip 只影響「目前焦點」的那一欄）。
   - 仍可點 input 本身選其他時間。

**優點**

- 改動集中在「加按鈕、Chip 與一點 JS」，不動現有 date/time 邏輯，風險低。
- 研究指出：原生 date/time 在許多手機上會開系統選擇器，觸控體驗已優化；加上「今天／明天」與常用時段 Chip，可明顯減少「難用」感。
- 與你們實務（14小陞、19小琦）一致，常用時段一鍵選。

**實作要點**

- 新增行程頁（add-event）與行程詳情編輯（event-detail）的**同一區塊**都加：日期旁「今天」「明天」、時間區上方/下方 Chip。
- 若目前有自訂 datePicker/timePicker 覆蓋原生，可改為**不綁定**這四個 input，讓它們用原生；或僅在「其他時間」時才開自訂滾輪。

**實作量**：小。

## 四、方案 2：日期改月曆點選、時間改 Chip +「其他」

**對象**：同樣四個欄位，但徹底改互動方式。

**做法**

1. **開始日期／結束日期**
   - 點擊欄位時**不再開滾輪**，改為彈出**月曆**（一頁一個月，可切換年月），使用者**點選某一天**即寫入該欄位。
   - 月曆下方可保留「今天」「明天」快捷，點一下選定並關閉彈窗。

2. **開始時間／結束時間**
   - 不先顯示滾輪，改為一排 **Chip**：`09:00`、`10:00`、`14:00`、`19:00`、`20:00`（可擴充），點選即寫入對應欄位。
   - 另提供 **「其他時間」** 按鈕，點開後再顯示現有時間滾輪或 `<input type="time">`，供精確選擇。

**優點**

- UX 研究一致建議：排程用**月曆點選**比滾輪直覺；時間用 **Chip** 可減少滑動、降低錯誤。
- 四個欄位仍在同一區塊，但操作從「多次滑動對齊」變成「點日期 + 點時段」。

**實作要點**

- 需實作或引入**輕量月曆元件**（僅需點選單日、切換月份即可）。
- 時間區 UI：Chip 區 +「其他時間」展開區（內為現有 time 滾輪或 input）。

**實作量**：中～大。

---

## 五、方案 3：Flatpickr + 手機用原生、表單加快捷與 Chip

**對象**：同樣四個欄位，用同一套程式兼顧手機與桌面。

**做法**

1. 引入 **Flatpickr**，將「開始日期」「結束日期」綁定 Flatpickr；**開始時間」「結束時間」** 可用 Flatpickr 的 time 或維持 `<input type="time">`。
2. Flatpickr 在**偵測到手機**時會自動改用**原生 date/time 選擇器**，在桌面上則用月曆與時間 UI，一碼兩用。
3. 表單上同樣加上**「今天」「明天」**與**時間 Chip**（09／10／14／19／20），作為輔助，減少必須打開選擇器的次數。

**優點**

- 手機端多由系統提供選擇器，觸控體驗較一致；桌面端有月曆可點。
- 若未來要限制可選日期、或需要 range 選範圍，Flatpickr 都支援。

**注意**

- 需確認在 LINE 內建瀏覽器與你們目標機型上的表現；若有異常可設 `disableMobile: true` 改回自訂 UI。
- 實作量略高於方案 1（引入與綁定、樣式）。

**實作量**：中。

---

## 六、建議選擇與下一步

- **想盡快改善、改動最小**：選 **方案 1**（原生 + 今天／明天 + 時間 Chip）。
- **願意做較大改版、希望完全不用滾輪**：選 **方案 2**（月曆 + Chip +「其他時間」）。
- **希望手機／桌面一套程式、且接受引入套件**：選 **方案 3**（Flatpickr + 快捷與 Chip）。
