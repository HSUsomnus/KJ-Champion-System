# 財力上傳權限說明

## 權限分級

系統現在支援三級權限：

### 1. 👑 開發者（Admin）
- **識別方式**：在 `.env` 檔案的 `ADMIN_LINE_USER_IDS` 中設定
- **權限**：
  - ✅ 可以查看**所有人**的財力上傳
  - ✅ 可以對**所有人**的財力留評語
  - ✅ 最高權限

### 2. 📋 負責人（Manager）
- **識別方式**：資料庫 `members` 表的 `role` 欄位設為 `'manager'`
- **權限**：
  - ✅ 可以查看**所有下級**的財力上傳
  - ✅ 可以對**所有下級**的財力留評語
  - ❌ 不能查看同級或上級的財力

### 3. 👥 上級（Superior）
- **識別方式**：透過 `invited_by` 關係（邀請人）
- **權限**：
  - ✅ 可以查看**直接下級**的財力上傳
  - ❌ **不能留評語**（只能查看）
  - ❌ 不能查看其他人的財力

### 4. 👤 一般成員（Member）
- **識別方式**：資料庫 `members` 表的 `role` 欄位為 `'member'`（預設值）
- **權限**：
  - ✅ 只能查看和上傳**自己**的財力
  - ❌ 不能查看他人的財力
  - ❌ 不能留評語

## 如何設定負責人

### 方法 1：直接在 Supabase 資料庫中修改

1. 登入 Supabase
2. 進入 `members` 表
3. 找到要設為負責人的成員
4. 將該成員的 `role` 欄位改為 `'manager'`
5. 儲存

### 方法 2：使用 SQL 指令

```sql
-- 將特定成員設為負責人
UPDATE members 
SET role = 'manager' 
WHERE line_id = '目標成員的LINE_USER_ID';

-- 查詢所有負責人
SELECT name, line_id, role FROM members WHERE role = 'manager';

-- 將負責人改回一般成員
UPDATE members 
SET role = 'member' 
WHERE line_id = '目標成員的LINE_USER_ID';
```

## 權限檢查流程

當用戶 A 嘗試查看用戶 B 的財力時：

```
1. 是否為開發者？
   → YES: 允許查看 + 允許留評語 ✅
   → NO: 繼續

2. 是否為負責人？
   → YES: 允許查看 + 允許留評語 ✅
   → NO: 繼續

3. 是否為 B 的上級？
   → YES: 允許查看 + 禁止留評語 ⚠️
   → NO: 拒絕存取 ❌
```

## 實際應用場景

### 場景 1：開發者
- 開發者可以查看並評論所有成員的財力
- 用於系統管理和全局監控

### 場景 2：負責人（業務主管）
- 負責人可以查看並評論團隊成員的財力
- 用於業務管理和財力審核
- 可以留下審核意見和建議

### 場景 3：上級（推薦人）
- 推薦人可以查看自己邀請的下級的財力
- **只能查看，不能留評語**
- 用於了解下級狀況，但不具審核權限

### 場景 4：一般成員
- 只能管理自己的財力上傳
- 可以看到負責人或開發者留下的評語
- 自己無法編輯評語

## 注意事項

1. **role 欄位值必須精確**：
   - 負責人：`'manager'`
   - 一般成員：`'member'`
   - 大小寫敏感

2. **ADMIN_LINE_USER_IDS 設定**：
   - 在 `.env` 檔案中設定
   - 多個 ID 用逗號分隔
   - 例如：`ADMIN_LINE_USER_IDS=U7cb2e1bb3b1a2302e16dded39287c144,Uxxxxx`

3. **上下級關係**：
   - 由 `invited_by` 欄位決定
   - 透過邀請連結自動建立
   - 不能留評語，只能查看

4. **權限生效時間**：
   - 修改資料庫後立即生效
   - 用戶重新進入頁面即可看到權限變更
