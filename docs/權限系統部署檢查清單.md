# 權限系統部署檢查清單

如果前端沒看到變化，請按照以下步驟逐一檢查：

## ✅ Step 1: 確認資料庫 Migration 已執行

### 1.1 檢查 role 欄位是否存在
到 **Supabase** → **SQL Editor**，執行：

```sql
SELECT column_name, data_type, column_default
FROM information_schema.columns
WHERE table_name = 'members' AND column_name = 'role';
```

**預期結果**：應該回傳一行資料
```
column_name | data_type         | column_default
role        | character varying | '一般人'::character varying
```

**如果沒有回傳結果**：代表 `role` 欄位還沒建立，請執行：
- 到 Supabase SQL Editor
- 執行 `database/add-role-column.sql` 的內容

---

### 1.2 設定自己為開發者
執行以下 SQL：

```sql
-- 1. 查詢所有成員（找到自己的 LINE ID）
SELECT line_id, name, role, created_at
FROM members
ORDER BY created_at DESC;

-- 2. 設定自己為開發者（替換成你的 LINE ID）
UPDATE members 
SET role = '開發者' 
WHERE name = '你的名字';  -- 或使用 line_id = 'U...'

-- 3. 驗證
SELECT name, role FROM members WHERE role = '開發者';
```

---

## ✅ Step 2: 確認 Vercel 部署狀態

### 2.1 檢查 Vercel Dashboard
1. 進入 https://vercel.com/dashboard
2. 選擇你的專案
3. 查看 **Deployments** 頁面

**檢查項目**：
- ✅ 最新的部署狀態是「Ready」（綠色勾勾）
- ✅ 部署時間是最近的（剛剛推送後的幾分鐘內）
- ❌ 如果顯示「Building」：等待建置完成（約 1-2 分鐘）
- ❌ 如果顯示「Error」：點進去查看錯誤訊息

### 2.2 查看建置日誌
如果部署失敗，點選該部署 → **Building** → 查看錯誤訊息

**常見錯誤**：
- 語法錯誤
- 缺少相依套件
- 環境變數缺失

---

## ✅ Step 3: 清除瀏覽器快取

### 3.1 強制重新整理
- **Windows/Linux**: `Ctrl + Shift + R` 或 `Ctrl + F5`
- **Mac**: `Cmd + Shift + R`

### 3.2 清除快取並硬性重新載入
1. 打開 LIFF App
2. 按 `F12` 開啟開發者工具
3. 在重新整理按鈕上**按右鍵**
4. 選擇「**清除快取並硬性重新載入**」

---

## ✅ Step 4: 測試權限 API

### 4.1 在瀏覽器 Console 測試
打開 LIFF App → 按 `F12` → Console，執行：

```javascript
// 取得你的 LINE User ID
liff.getProfile().then(p => console.log('LINE ID:', p.userId))

// 測試權限 API（替換成你的 LINE ID）
fetch('/api/admin/check?userId=你的LINE_ID')
  .then(r => r.json())
  .then(d => console.log('權限檢查結果:', d))
```

**預期結果**：
```json
{
  "success": true,
  "data": {
    "isAdmin": true,
    "role": "開發者"
  }
}
```

**如果出現錯誤**：
- `role column does not exist` → 資料庫 migration 未執行
- `403 Forbidden` → 權限不足，檢查資料庫中的 role 欄位
- `500 Internal Server Error` → 後端錯誤，查看 Vercel Logs

---

## ✅ Step 5: 查看 Vercel Runtime Logs

1. Vercel Dashboard → 你的專案
2. 點選 **Deployments** → 選擇最新的部署
3. 點選 **Functions** 或 **Runtime Logs**
4. 查找是否有錯誤訊息

**常見錯誤日誌**：
```
Error: column "role" does not exist
```
→ 資料庫 migration 未執行

---

## 🎯 快速診斷指令

在 Supabase SQL Editor 執行以下「一鍵診斷」：

```sql
-- 診斷檢查
SELECT 
  '資料庫 role 欄位' AS 檢查項目,
  CASE 
    WHEN COUNT(*) > 0 THEN '✅ 已建立'
    ELSE '❌ 未建立（請執行 add-role-column.sql）'
  END AS 狀態
FROM information_schema.columns
WHERE table_name = 'members' AND column_name = 'role'

UNION ALL

SELECT 
  '開發者數量' AS 檢查項目,
  CASE 
    WHEN COUNT(*) > 0 THEN '✅ 已設定 ' || COUNT(*) || ' 位開發者'
    ELSE '❌ 尚未設定開發者（請執行 UPDATE members SET role = ''開發者'' WHERE ...）'
  END AS 狀態
FROM members
WHERE role = '開發者';
```

## 📝 完整設定流程（如果都還沒做）

### 1️⃣ 資料庫設定
```sql
-- 執行 add-role-column.sql（新增 role 欄位）
-- 然後設定自己為開發者
UPDATE members SET role = '開發者' WHERE name = '你的名字';
```

### 2️⃣ 等待 Vercel 部署完成
- 查看 Vercel Dashboard 確認部署狀態為「Ready」

### 3️⃣ 清除快取
- `Ctrl + Shift + R` 強制重新整理

### 4️⃣ 測試
- 進入 LIFF App → 個人資料頁
- 應該會看到：
  - 顯示「您的權限：開發者」
  - 紫色的「開發人員專用功能」區塊
  - 兩個按鈕：「👥 權限管理」和「🎂 強制同步所有用戶生日行程」
