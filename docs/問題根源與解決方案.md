# 問題根源與解決方案

## 🔍 問題：為什麼看不到新功能？

### 根本原因
這個專案有**兩套前端**，而 Vercel 只部署了舊的那套！

1. **舊前端** (`public/` 目錄)
   - 純 HTML + JavaScript
   - ✅ **這是你目前看到的版本**
   - 檔案：`public/profile.html`, `public/js/profile.js`
   
2. **新前端** (`frontend/` 目錄)
   - React + Vite
   - ❌ **沒有被建置和部署**
   - 檔案：`frontend/src/pages/ProfilePage.jsx`
   - 🎯 **所有新功能都在這裡！**

### 為什麼會這樣？
之前的 `vercel.json` 配置：
```json
{
  "builds": [{"src": "api/index.js", "use": "@vercel/node"}],
  "routes": [
    {
"src": "/api/(.*)", 
      "dest": "/api/index.js"
    },
    {
      "src": "/(.*)",
      "dest": "/api/index.js"  // 所有請求都導向後端
    }
  ]
}
```

而 `server/server.js` 只提供 `public/` 的靜態檔案：
```javascript
const publicPath = path.join(__dirname, '../public');
app.use(express.static(publicPath));
```

所以 React 前端（`frontend/`）**完全沒有被建置**！

---

## ✅ 解決方案（已修正）

### 1. 修改 `vercel.json`
新增 buildCommand 和 outputDirectory：

```json
{
  "buildCommand": "cd frontend && npm install && npm run build",
  "outputDirectory": "frontend/dist",
  "routes": [
    {"src": "/api/(.*)", "dest": "/api/index.js"},
    {"handle": "filesystem"},
    {"src": "/(.*)", "dest": "/index.html"}
  ]
}
```

### 2. Vercel 部署流程（現在會這樣運作）
1. ✅ 執行 `cd frontend && npm install && npm run build`
2. ✅ 建置 React 前端到 `frontend/dist/`
3. ✅ 將 `frontend/dist/` 設為靜態檔案根目錄
4. ✅ API 請求導向 `api/index.js`
5. ✅ 其他請求導向 React 的 `index.html`（SPA 路由）

## 🎯 如果還是看到舊版前端

### 可能原因 1：Vercel 建置失敗
**檢查**：
- Vercel Dashboard → Deployments → 點選最新部署
- 查看 Build Logs 是否有錯誤

**常見錯誤**：
```
Error: Command "cd frontend && npm install && npm run build" exited with 1
```

**解決**：
- 可能是 buildCommand 語法不被 Vercel 支持
- 需要在 Vercel Dashboard 手動配置

### 可能原因 2：快取問題
**解決**：
1. 完全關閉 LINE App
2. 清除 LINE 快取（設定 → 聊天 → 清除快取）
3. 重新開啟

### 可能原因 3：LIFF Endpoint URL 指向錯誤
**檢查**：
- LINE Developers Console
- 你的 LIFF App 設定
- Endpoint URL 是否正確
