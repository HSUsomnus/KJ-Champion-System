# 發布前機密檢查清單

推送到 GitHub 並觸發 Cloud Run 前，請確認**沒有任何金鑰、ID、私鑰**被 commit。

---

## 一、絕對不能進 Git 的檔案／資料夾

| 項目 | 說明 | .gitignore 是否已包含 |
|------|------|------------------------|
| `.env` | 環境變數（私鑰、Token、Calendar ID、LIFF ID 等） | ✅ `.env`、`.env.local`、`.env.*.local` |
| `Key/` | Google Service Account JSON、任何金鑰檔 | ✅ `Key/` |
| `*.json`（除 package） | 金鑰 JSON 檔 | ✅ `*.json`，並排除 `package.json`、`package-lock.json` |

---

## 二、程式碼裡不能出現的內容

- **不要**在 `.js`、`.html`、`.yaml` 等原始碼中寫死：
  - Google Calendar ID（例如 `xxx@group.calendar.google.com`）
  - Google Sheets ID
  - LINE LIFF ID、Channel ID、Channel Secret、Channel Access Token
  - LINE 加好友連結（例如 `https://line.me/R/ti/p/@實際帳號`）
  - 私鑰（`-----BEGIN PRIVATE KEY-----` 整段）
  - ngrok 或任何「你的」對外網址（例如 `proconvention-jani-fledgy.ngrok-free.dev`）
- 上述皆應來自 **環境變數**（`.env` 或 Cloud Run 環境變數），不寫死在程式碼。

---

## 三、推上前請在本機執行

1. **確認 .env 不會被 commit**
   ```bash
   git status
   ```
   - 清單中**不應**出現：`.env`、`Key/`、任何 `.json`（除 package.json / package-lock.json）。

2. **若曾誤 add 過 .env 或 Key/**
   ```bash
   git reset HEAD .env
   git reset HEAD Key/
   ```
   再確認 `git status` 沒有這些檔案。

3. **搜尋是否有殘留機密**（可選）
   - 在專案中搜尋你的「實際」Calendar ID、Sheets ID、LIFF ID、Token 前幾字。
   - 只應出現在 `.env`（而 `.env` 已被 .gitignore），不應出現在任何會被 commit 的檔案。

---

## 四、Cloud Run 環境變數（發布後）

金鑰不放在程式碼裡，改在 **Cloud Run 服務** 的「環境變數」或 **Secret Manager** 設定：

- `GOOGLE_SERVICE_ACCOUNT_EMAIL`
- `GOOGLE_PRIVATE_KEY`
- `GOOGLE_PROJECT_ID`
- `GROUP_CALENDAR_ID`
- `MEMBER_SHEET_ID`、`MEMBER_SHEET_NAME`
- `LIFF_ID`、`LINE_CHANNEL_ID`、`LINE_CHANNEL_SECRET`
- `LINE_CHANNEL_ACCESS_TOKEN`
- `APP_URL`（正式網址，例如 Cloud Run 的 URL）

詳見根目錄 `DEPLOYMENT.md`。

---

## 五、已修正的項目（本次發布前）

- `server/services/lineService.js` 已移除硬編碼的 **GROUP_CALENDAR_ID** 與 **LINE 加好友連結**，改由環境變數提供。

---

**總結：推上 GitHub 的只有「程式碼」；金鑰與 ID 只放在本機 `.env`、`Key/`（不推），或 Cloud Run／Secret Manager。**
