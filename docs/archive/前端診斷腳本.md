# 前端診斷腳本

## 🔍 在瀏覽器 Console 執行（按 F12）

進入 LIFF App 後，按 `F12` 開啟 Console，然後逐一執行以下指令：

---

### 1️⃣ 取得你的 LINE User ID
```javascript
liff.getProfile().then(p => {
  console.log('=== LINE 個人資料 ===');
  console.log('User ID:', p.userId);
  console.log('Display Name:', p.displayName);
  console.log('請複製上面的 User ID');
});
```

---

### 2️⃣ 測試權限 API（替換成你的 User ID）
```javascript
const userId = 'U你的USER_ID'; // 替換成步驟1取得的 User ID

fetch(`/api/admin/check?userId=${userId}`)
  .then(res => {
    console.log('=== API 回應狀態 ===');
    console.log('Status:', res.status);
    console.log('OK:', res.ok);
    return res.json();
  })
  .then(data => {
    console.log('=== API 回應內容 ===');
    console.log('Success:', data.success);
    console.log('Is Admin:', data.data?.isAdmin);
    console.log('Role:', data.data?.role);
    console.log('完整資料:', JSON.stringify(data, null, 2));
  })
  .catch(err => {
    console.error('=== API 錯誤 ===');
    console.error(err);
  });
```

**預期結果（如果一切正常）**：
```
=== API 回應狀態 ===
Status: 200
OK: true
=== API 回應內容 ===
Success: true
Is Admin: true
Role: 開發者
```

**如果出現錯誤**：
- `Status: 500` → 後端錯誤（可能資料庫欄位不存在）
- `Status: 401` → 權限驗證失敗
- `Status: 404` → API 路由不存在

---

### 3️⃣ 測試取得個人資料
```javascript
const userId = 'U你的USER_ID'; // 替換成你的 User ID

fetch(`/api/profile?userId=${userId}`)
  .then(res => res.json())
  .then(data => {
    console.log('=== 個人資料 ===');
    console.log('Name:', data.data?.name);
    console.log('Role:', data.data?.role);
    console.log('完整資料:', JSON.stringify(data, null, 2));
  })
  .catch(err => {
    console.error('錯誤:', err);
  });
```

**預期結果**：
```
=== 個人資料 ===
Name: 你的名字
Role: 開發者
```

**如果 Role 是 undefined**：
→ 資料庫中的 role 欄位沒有值，請執行：
```sql
UPDATE members SET role = '開發者' WHERE name = '你的名字';
```

---

### 4️⃣ 檢查前端 React State
在個人資料頁面，開啟 React DevTools（F12 → Components），找到 `ProfilePage` 組件，查看 State：
- `isAdmin` 應該是 `true`
- `userRole` 應該是 `'開發者'`

---

### 5️⃣ 模擬完整的個人資料載入流程
```javascript
const userId = 'U你的USER_ID'; // 替換成你的 User ID

console.log('=== 開始測試完整流程 ===');

// 步驟 1: 取得個人資料
fetch(`/api/profile?userId=${userId}`)
  .then(res => res.json())
  .then(profileData => {
    console.log('✅ 步驟 1: 取得個人資料');
    console.log('  Name:', profileData.data?.name);
    console.log('  Role:', profileData.data?.role);
    
    // 步驟 2: 取得權限等級
    return fetch(`/api/admin/check?userId=${userId}`)
      .then(res => res.json())
      .then(roleData => {
        console.log('✅ 步驟 2: 取得權限等級');
        console.log('  Is Admin:', roleData.data?.isAdmin);
        console.log('  Role:', roleData.data?.role);
        
        // 模擬前端邏輯
        const isAdmin = roleData.data?.isAdmin || false;
        const userRole = roleData.data?.role || profileData.data?.role || '一般人';
        
        console.log('✅ 步驟 3: 前端 State 應該設定為:');
        console.log('  isAdmin:', isAdmin);
        console.log('  userRole:', userRole);
        
        // 檢查顯示邏輯
        console.log('✅ 步驟 4: 顯示邏輯檢查');
        console.log('  userRole 存在?', !!userRole);
        console.log('  應該顯示權限區塊?', !!userRole);
        console.log('  應該顯示開發人員區塊?', isAdmin);
        
        return { isAdmin, userRole };
      });
  })
  .then(result => {
    console.log('=== 測試完成 ===');
    console.log('最終結果:', result);
  })
  .catch(err => {
    console.error('❌ 測試失敗:', err);
  });
```

## 🐛 常見問題與解決方案

### 問題 1: API 回應 `role: null` 或 `role: undefined`
**原因**：資料庫中沒有設定 role
**解決**：
```sql
-- 在 Supabase SQL Editor 執行
UPDATE members SET role = '開發者' WHERE line_id = 'U你的USER_ID';
```

### 問題 2: API 回應 `column "role" does not exist`
**原因**：資料庫沒有執行 migration
**解決**：執行 `database/add-role-column.sql`

### 問題 3: 前端顯示 `userRole: "一般人"`
**原因**：API 正常，但資料庫中的 role 是「一般人」
**解決**：
```sql
UPDATE members SET role = '開發者' WHERE line_id = 'U你的USER_ID';
```

### 問題 4: Console 有紅色錯誤訊息
**可能原因**：
- 網路錯誤
- CORS 錯誤
- API 路由不存在

**解決**：
1. 確認 Vercel 部署成功
2. 清除快取並重新載入（Ctrl + Shift + R）
3. 檢查 Vercel Logs 是否有錯誤
