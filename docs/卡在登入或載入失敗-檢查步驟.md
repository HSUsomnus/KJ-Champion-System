# 卡在「導向登入中…」或「載入失敗」檢查步驟

當 LIFF 行事曆卡在登入畫面或列表顯示「載入失敗, 請稍後再試」時，可依下面順序檢查。

---

## 一、卡在「導向登入中…」

按了「使用 LINE 登入」後一直停在「導向登入中…」，多半是 **LIFF 設定** 或 **網址沒對上**。

### 1. 確認是在「LINE App 內」開網頁

- LIFF 登入（`liff.login()`）是設計給 **LINE 內建瀏覽器** 用的。
- 若是在 Chrome、Safari 等一般瀏覽器直接開 `line-liff-calendar.vercel.app`，登入導向可能異常或卡住。
- **做法**：在 LINE 裡從「聊天室／圖文選單／Bot 發的連結」點進你的 LIFF 網址再試一次。

### 2. 確認 LIFF Endpoint URL 與網域一致

- 到 [LINE Developers Console](https://developers.line.biz/) → 你的 Channel → **LIFF** 分頁 → 點你的 LIFF App。
- **Endpoint URL** 必須是實際放 LIFF 的網址，例如：  
  `https://line-liff-calendar.vercel.app/`  
  （結尾要不要加 `/` 依你實際設定為準，但要和 Vercel 上的網址一致。）
- 若這裡填錯或還是舊網址，LINE 登入後可能不會正確導回你的頁面，看起來就像卡在「導向登入中…」。

### 3. 確認 Vercel 環境變數有 LIFF_ID

- Vercel 專案 → **Settings** → **Environment Variables**。
- 要有 **LIFF_ID**，值從 LINE Developers → LIFF 分頁的 **LIFF ID** 複製貼上。
- 沒設或設錯，前端會拿不到 LIFF ID，登入流程也會異常。

---

## 二、行程列表「載入失敗, 請稍後再試」

列表是呼叫後端 **`/api/calendar/month`**，後端再向 **Supabase（PostgreSQL）** 要資料。載入失敗通常是 **API 錯誤** 或 **資料庫連不到**。

### 1. 確認 Vercel 上有設 DATABASE_URL

- 後端用 **DATABASE_URL** 連 Supabase（見 `server/config/db.js`）。
- Vercel → **Settings** → **Environment Variables** → 要有 **DATABASE_URL**。
- 值從 Supabase → **Project Settings** → **Database** → **Connection string**（選 **URI**，Transaction pooler 的即可）。
- 儲存後記得 **重新部署** 一次，環境變數才會生效。

### 2. 檢查 Supabase 是否可連、表是否存在

- 到 Supabase → **Table Editor**，確認有 **events**、**members** 等表。
- 若已啟用 RLS，要確認有下過 **database/rls-policies.sql** 的政策，否則若用 anon 等角色直連可能會被擋（你目前是後端用 DATABASE_URL 連線，通常不受 RLS 影響，但可一併確認）。

### 3. 看 Vercel 的 API 日誌

- Vercel 專案 → **Deployments** → 點最新一次部署 → **Functions**，或從 **Logs** 看執行紀錄。
- 請求 `/api/calendar/month` 時若有 **500** 或連線錯誤，日誌裡會出現對應錯誤訊息（例如 `connection refused`、`DATABASE_URL` 未設定等），可依此修正。

---

## 三、Supabase Security Advisor 顯示「RLS Disabled in Public」

這不會直接造成「載入失敗」（你的後端是用 DATABASE_URL 直連，通常會繞過 RLS），但建議修掉以通過安全檢查：

1. 在專案裡已準備好 **database/rls-policies.sql**。
2. 到 Supabase → **SQL Editor** → 貼上該檔內容 → **Run**。
3. 執行完到 **Advisors** → **Security Advisor** → 點 **Rerun linter**，三個 RLS 錯誤應會消失。

---

## 快速對照表

| 狀況 | 優先檢查 |
|------|----------|
| 卡在「導向登入中…」 | 1) 是否在 LINE App 內開啟 2) LIFF Endpoint URL 是否為 Vercel 網址 3) Vercel 是否有 LIFF_ID |
| 行程列表「載入失敗」 | 1) Vercel 是否設 DATABASE_URL 2) 部署後有無重新 deploy 3) Vercel Functions 日誌錯誤訊息 |
| Security Advisor RLS 錯誤 | 執行 database/rls-policies.sql，再 Rerun linter |

完成上述設定後，再從 LINE App 內重新打開 LIFF 測試一次即可。
