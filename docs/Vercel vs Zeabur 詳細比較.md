# Vercel vs Zeabur 詳細比較

> 適用情境：LINE LIFF 應用（含成員資料、行事曆、後端 API、資料庫）
> 更新日期：2025 年 2 月

---

## 一、費用比較

### 1.1 入門費用

| 項目 | Vercel | Zeabur |
|------|--------|--------|
| **免費方案** | Hobby 方案，永久免費 | 免費試用，每月 $5 額度 |
| **最低付費門檻** | Pro：$20/月起 | 開發者：$5/月起 |
| **需綁信用卡** | 免費方案不需 | 免費試用不需 |
| **付費後含額度** | Pro 含 $20 用量額度 | 開發者含 $5 用量額度 |

### 1.2 資源計費方式

**Vercel（用量計費，超出額度後加價）**
- CPU：約 $0.128 / 小時（Fluid Compute）
- 記憶體：約 $0.0106 / GB-小時
- Function 呼叫：100 萬次內免費，之後約 $0.60 / 100 萬次
- 網路輸出：100 GB 免費，之後約 $0.15 / GB

**Zeabur（按分鐘計費）**
- CPU：$0 / vCPU-分鐘（目前免費）
- 記憶體：約 $0.00025 / GB-分鐘（約 $10 / GB / 月）
- 輸出流量：$0.1 / GB
- 持久儲存：$0.2 / GB

### 1.3 資料庫費用

| 項目 | Vercel | Zeabur |
|------|--------|--------|
| **內建 Postgres** | 需透過 Marketplace 接 Neon | 內建 PostgreSQL、MySQL 等 |
| **Neon 免費額度** | 0.5 GB 儲存、100 CU-小時 | — |
| **Zeabur 資料庫** | — | 依記憶體、儲存空間計費 |

### 1.4 小專案預估月費（LIFF + 後端 + 輕量 DB）

| 情境 | Vercel | Zeabur |
|------|--------|--------|
| **極輕量（個人／測試）** | $0（Hobby + Neon 免費） | $0（$5 額度內） |
| **小型團隊（約 10 人用）** | $0～$20 | $5～$15 |
| **用量較大（多人同時）** | $20～$50+ | $5～$30 |

**小結**：Zeabur 入門門檻較低（$5 vs $20），輕量專案成本通常較省；Vercel 在免費額度內可做到 $0。

## 二、延遲與效能

### 2.1 Cold Start（冷啟動）

| 項目 | Vercel | Zeabur |
|------|--------|--------|
| **部署模型** | Serverless（無伺服器） | Container（常駐容器） |
| **冷啟動** | 有，閒置一段時間後首次請求可能較慢（約 0.5～3 秒） | 通常無，容器持續運行 |
| **Fluid Compute** | Pro 方案可降低冷啟動 | — |
| **Bytecode 快取** | Node.js 20+ 有，可加速約 60% | — |

### 2.2 回應延遲（台灣使用者）

| 項目 | Vercel | Zeabur |
|------|--------|--------|
| **節點位置** | 全球 CDN，亞洲有節點 | 可選 GCP 台灣機房 |
| **台灣延遲** | 約 50～150 ms（視節點） | 台灣機房約 20～50 ms |
| **資料庫連線** | Neon 可能在不同區域 | DB 與 App 可在同區域，延遲較低 |

**小結**：Zeabur 若選台灣機房，對台灣使用者延遲較佳；Vercel 依 CDN 節點，一般也夠用。

## 三、穩定性

### 3.1 運行模式

| 項目 | Vercel | Zeabur |
|------|--------|--------|
| **架構** | Serverless，按請求啟動 | Container，持續運行 |
| **流量突增** | 自動擴展，理論上較能應付 | 需預留資源或自動擴展 |
| **長時間無流量** | 可能進入休眠 → 冷啟動 | 容器可能縮放，取決設定 |
| **SLA** | Pro 無明確 SLA；Enterprise 有 99.99% | 依方案不同 |

### 3.2 華語區存取（台灣／中國）

| 項目 | Vercel | Zeabur |
|------|--------|--------|
| **台灣** | 一般可正常存取 | 可選台灣機房，較穩定 |
| **中國大陸** | 預設網域可能有 DNS 汙染，較難存取 | 依節點而定，通常需自訂網域或 CDN |
| **自訂網域** | 支援，可改善中國存取 | 支援 |

**小結**：LIFF 主要在 LINE 環境，多數流量來自台灣；兩者在台灣都算穩定，Vercel 在中國需額外 DNS 設定。

## 四、設定與操作難易度

### 4.1 介面與文件語言

| 項目 | Vercel | Zeabur |
|------|--------|--------|
| **後台介面** | 英文為主 | 支援繁體中文 |
| **官方文件** | 英文 | 繁體中文、簡體中文 |
| **技術支援** | 英文社群、Discord | 中文 Discord、論壇 |

### 4.2 部署流程

| 項目 | Vercel | Zeabur |
|------|--------|--------|
| **連結 GitHub** | 一鍵部署，自動 CI/CD | 一鍵部署，自動 CI/CD |
| **框架辨識** | 自動辨識 Next.js、Vite 等 | 自動辨識多種框架 |
| **環境變數** | 在後台設定 | 在後台設定 |
| **自訂 Dockerfile** | 支援有限 | 完整支援 |

### 4.3 資料庫設定

| 項目 | Vercel | Zeabur |
|------|--------|--------|
| **新增資料庫** | 需到 Marketplace 裝 Neon 等 | 內建一鍵新增 PostgreSQL、MySQL |
| **連線字串** | 需手動複製到環境變數 | 可自動注入環境變數 |
| **前後端 + DB 整合** | 分開設定 | 同一專案內一次完成 |

**小結**：Zeabur 對中文使用者較友善，資料庫整合較直覺；Vercel 生態成熟，但需習慣英文介面與額外整合步驟。

## 六、總結與建議

### 選 Vercel 較適合的情境
- 已有 Vercel 經驗或偏好 Next.js 生態
- 希望完全免費起步（Hobby + Neon 免費額度）
- 流量以全球為主，不特別要求台灣最低延遲
- 能接受英文介面與文件

### 選 Zeabur 較適合的情境
- 需要完整後端 + 資料庫，希望一次搞定
- 預算有限，希望入門門檻低（$5/月）
- 重視繁體中文介面與支援
- 使用者主要在台灣，想降低延遲

### 針對你的 LINE LIFF 專案
- **若已用 Vercel**：可繼續用 Vercel，搭配 Neon 做成員與行事曆資料庫遷移，或用現有成員快取方案先撐住。
- **若考慮整體遷移**：Zeabur 在「後端 + 資料庫 + 中文支援」上較省事，費用門檻也較低。
