# 成員資料試算表欄位說明

專案使用 **Google 試算表** 的「成員資料」工作表儲存成員資料，程式碼會依欄位順序讀寫，請依下表設定**第一列（標題列）**與資料。

---

## 欄位對照表

| 欄位 | 程式碼對應 | 標題建議（第一列） | 說明 |
|------|------------|--------------------|------|
| **A** | `lineId` | LINE ID | LINE User ID（必填） |
| **B** | `name` | 姓名 | 真實姓名 |
| **C** | `email` | Email | 電子郵件 |
| **D** | `phone` | 電話 | 電話號碼 |
| **E** | `starLevel` | 星等 | 白星／綠星／橙星／紅星／紫星 |
| **F** | `courseRecord` | 課程紀錄 | 課程紀錄文字（複選：正式金流課、財富藍圖課等） |
| **G** | `pictureUrl` | **頭像URL** | LINE 頭像網址（成員列表／詳情顯示用） |
| **H** | `teslaFranchisee` | **是否為特斯拉出行加盟主** | 下拉：`是` 或 `否`（進階資訊） |
| **I** | `teamResponsibilities` | **團隊負責事項** | 文字，自行輸入（進階資訊） |
| **J** | `volunteerRecords` | **課程志工紀錄** | JSON 字串，格式如 `[{"date":"2025-02-01","option":"金流"}]`（進階資訊） |

---

## G 欄（頭像URL）對應程式碼要求

- **第一列標題請填：`頭像URL`**（或英文 `pictureUrl`，程式只看第 7 欄位置，標題不影響讀取）。
- 程式碼會讀取 **第 7 欄（G 欄）** 當作 `pictureUrl`，用來在成員列表與成員詳情顯示 LINE 頭像。
- 資料可由**註冊**或**更新個人資料**時自動寫入；若尚未有 G 欄，請在試算表新增一欄，第一列填 **「頭像URL」** 即可。

---

## 注意事項

- 第一列為標題列，程式從 **第二列（A2:J）** 開始讀取資料。
- G 欄可留空；留空時成員列表與詳情會顯示預設頭像圖。
- 已存在的成員可到 LIFF「個人」頁面按一次「更新個人資料」，系統會將目前 LINE 頭像 URL 寫入 G 欄。

---

## 進階資訊欄位（H、I、J）

- **H 欄（是否為特斯拉出行加盟主）**：第一列標題建議填 **「是否為特斯拉出行加盟主」**。程式接受 `是` 或 `否`，留空視為未填。
- **I 欄（團隊負責事項）**：第一列標題建議填 **「團隊負責事項」**。自由文字，可留空。
- **J 欄（課程志工紀錄）**：第一列標題建議填 **「課程志工紀錄」**。程式會存成 JSON 陣列字串，例如：`[{"date":"2025-02-01","option":"金流"},{"date":"2025-02-05","option":"藍圖"}]`。`option` 僅限「金流」或「藍圖」。若尚無紀錄可留空或存 `[]`。

**請在既有「成員資料」工作表中，於 G 欄右側新增 H、I、J 三欄**，第一列填上上述標題，程式會從 **A2:J** 讀寫資料。
