## 改動摘要（給 CHANGELOG 用）

團隊調查表單系統（KJ Survey）實作完成。副總／管理者發佈調查表單＝發佈任務，登入後台即時追蹤「誰完成／誰沒完成」。兩端不對稱：夥伴端免登入零打字、管理者端手機優先儀表板。

- **Section M 後端合併**：舊獨立 `kj-survey-server/` 搬入主系統 `server/`——路由 `server/routes/survey/*`（public 前台 / admin 後台 / requireAdminRole 閘門）、service `server/services/survey/*`，API 前綴 `/api/survey/*`，複用既有 pg pool，隨主後端一起部署（`_worker.js` 無需新增代理）。前端頁面搬入 `frontend/src/pages/survey/*`、`surveyApi.js` 路徑改 `/api/survey/*`、App.jsx 加 `/f/:token` 與 `/admin` 獨立 route。migration SQL 搬至 `server/migrations/001_init_survey_tables.sql`，`exceljs` 併入根依賴。
- **Section 4 儀表板**：後端 `computeAttendance`（全名單比對、按推薦人分組、各組進度、整體完成率）；前端側邊欄任務清單 + 首屏完成率 hero + 分組進度卡片（ProgressMeter + ✅/❎），SurveyAdmin 改手機優先（移除強制 width=1280）。
- **Section 5 明細 + 篩選**：後端 `listSubmissions`；前端 SubmissionsView + submissionFilter 純函式（單條件互斥：姓名→推薦人+本人、星等、課程欄 Yes），儀表板加「完成狀況/明細」切換。
- **Section 6 匯出**：後端 exportService（CSV 含 BOM + 逃逸、exceljs xlsx）；前端 CSV/Excel 下載按鈕（`requireAdminRole` 支援 `?lineUserId` query，因下載連結無法帶 header）。
- **Section 7 建立器**：後端 createForm/updateForm/publishForm + validateFields；前端 FormBuilder（欄位增刪、型態切換、發佈流程、複製連結）+ FormPreview 唯讀預覽。
- **Section 8**：README 新增 KJ Survey 專章、`_worker.js` SPA fallback 確認。

零主系統既有程式碼破壞（僅在 server.js 掛載 survey 路由、App.jsx 加 route）。

## 學習日誌素材（實作中遇到的坑）

1. **分支拓撲分裂**：策略修訂版 spec/tasks 在指定分支 `claude/chang20-implementation-gtdidf`，但已完成的 Section 1–3 程式碼在 `m_b_調查表單`（舊獨立 `kj-survey-server/` 結構），兩邊還各帶不同版本 spec/tasks。
   - 試過的方法：直接 merge m_b_調查表單 會拉回舊 spec + 一堆無關 `.claude` 歷史，污染分支。
   - 最終解法：用 AskUserQuestion 確認 → 使用者選「在指定分支搬碼實作」。用 `git checkout origin/m_b_調查表單 -- <前端檔>` 取前端頁面原封搬入，`git show` 讀舊後端當搬遷來源、就地改寫進 `server/`，不 merge 整條歷史。
   - 根本原因：CCR 自動開的 `claude/*` 分支與使用者手動 `m_b_*` 分支各自演進，沒有交會點。

2. **now.md 地雷過時**：`.claude/now.md` 記「CCR 沙箱 npm / node 不可用」，但本環境 `/opt/node22/bin` 有 node v22.22.2 + npm 10.9.7，`npm install` + `npx jest` / `npx vitest` 全可跑，蓋一層測一層得以在 CCR 內完成。→ 現象：地雷過時；建議收尾時提醒使用者複查此條。

3. **既有後端測試 3 suite 失敗（非本 change 造成）**：`npx jest` 全跑時 googleAuth / debug / calendarSync 三 suite 共 21 test 失敗——皆因 CCR 缺 Google SA 憑證 / DB / `GROUP_CALENDAR_ID` 回 503，與 survey 無關（三者皆未 import survey 或 server.js）。survey 8 個 suite 全 PASS。

4. **測試小坑**：
   - AttendanceView 中「曹琬琦」同時是組標題與另一組的成員 → getByText 重複，改 getAllByText。
   - 匯出下載用 `<a href>` 無法帶 header → surveyApi 產生 `?lineUserId=` query，`requireAdminRole` 用 `req.query?.lineUserId` fallback（optional chaining 避免 unit test 無 req.query 時炸）。
