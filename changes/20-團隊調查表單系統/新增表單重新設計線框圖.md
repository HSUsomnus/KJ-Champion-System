# Change 20 — 新增表單重新設計線框圖

> 狀態：**審查草案，尚未定案**
>
> 目的：重新設計「新增／編輯表單」體驗，操作模式參考 Google Forms，但視覺維持 KJ Champion System 的 Warm Minimal 設計系統。
>
> 本文件只呈現資訊架構、操作位置與互動狀態，不修改 `spec.md`、`tasks.md` 或產品程式碼。使用者確認後，才由規劃流程更新正式計畫。

---

## 一、設計原則

1. 編輯器改為單欄題目卡，不再用左右各半的常駐預覽。
2. 一次聚焦一題；選取中的題目展開完整編輯，其他題目顯示摘要。
3. 題型、答案內容與題目操作建立清楚層級。
4. 預覽改為獨立模式，使用與實際 `/f/:token` 相同的欄位 renderer。
5. 不照搬 Google Forms 紫色外觀，沿用 KJ：
   - 背景 `#F7F5F2`
   - 卡片 `#FFFFFF`
   - 主文字 `#2C2C2C`
   - 次文字 `#8A8680`
   - 強調色 `#4A7C59`
   - 邊框 `#E2DED8`
6. 第一版維持既有三種題型：
   - 文字
   - 搜尋選單
   - 是否
7. 不變更 DB schema、後端 API 與表單 JSON 的既有必要欄位。

## 二、整體流程

```text
調查表單後台
    │
    ├─ 點「＋ 新增」
    │
    ▼
新增表單編輯器
    │
    ├─ 編輯標題
    ├─ 新增／編輯／排序題目
    ├─ 儲存草稿
    ├─ 進入預覽
    │      └─ 返回編輯器
    └─ 發佈表單
           ├─ 發佈成功
           ├─ 複製填寫連結
           └─ 返回後台查看送出紀錄
```

### 3.2 頂部操作列

```text
┌──────────────────────────────────────────────────────────────────────────────────────┐
│ [← 返回表單列表]  新增表單                   未儲存  [眼睛 預覽] [儲存草稿] [發佈表單] │
└──────────────────────────────────────────────────────────────────────────────────────┘
```

| 元素 | 行為 |
|---|---|
| 返回表單列表 | 若內容有未儲存變更，先顯示離開確認 |
| 新增表單／表單名稱 | 顯示目前操作對象，不取代表單標題輸入 |
| 未儲存 | 狀態文字；儲存後改為「已儲存」 |
| 預覽 | 切換到完整預覽模式 |
| 儲存草稿 | 呼叫現有 create／patch API |
| 發佈表單 | 新表單尚未儲存、無題目或驗證失敗時禁用 |

狀態文字：

```text
未儲存        暖灰
儲存中…      暖灰
已儲存        墨綠
儲存失敗      錯誤紅
```

> 審查點：此版先保留「手動儲存草稿」，不直接導入 Google Forms 的自動儲存。

## 五、題目卡互動

### 5.1 未選取題目：摘要卡

```text
┌──────────────────────────────────────────────────────────────┐
│ ⋮⋮   姓名                                     搜尋選單       │
│      從成員名單搜尋或手動輸入                                │
└──────────────────────────────────────────────────────────────┘
```

規則：

- 點擊卡片任一空白處即可進入編輯。
- 摘要卡只顯示題目名稱、題型與答案形式摘要。
- 不顯示刪除、必填等操作，降低視覺噪音。
- `⋮⋮` 是排序把手；使用 inline SVG，不使用文字符號實作。

### 5.2 選取題目：完整編輯卡

```text
┌──────────────────────────────────────────────────────────────┐
│                            ⋮⋮                                │
│                                                              │
│  [ 問題標題                                      ] [題型  ▾] │
│                                                              │
│  ───────────── 依題型變化的答案編輯區 ─────────────          │
│                                                              │
│  [進階設定 ▾]                                                │
│                                                              │
├──────────────────────────────────────────────────────────────┤
│                      [複製] [刪除]  │  必填  [開關]           │
└──────────────────────────────────────────────────────────────┘
```

選取視覺：

- 左側 `4px` 墨綠色條。
- 邊框維持 `#E2DED8`，不使用厚綠框。
- 陰影只用既有卡片陰影。
- 展開／收合過渡不超過 `200ms`。

### 5.3 題型選單

```text
┌──────────────────────┐
│ 文字                 │
│ 搜尋選單             │
│ 是否                 │
└──────────────────────┘
```

規則：

- 題型名稱採使用者語言，不顯示 `text`、`searchable_select`、`yesno`。
- 題型變更可能清除既有選項時，需要二次確認。
- 下拉選單使用既有 surface／border 色，不導入第三方元件。


答案區只呈現填寫者將看到的單行文字樣貌，不讓管理者在此輸入答案。

### 6.2 搜尋選單：靜態選項

```text
┌──────────────────────────────────────────────────────────────┐
│                            ⋮⋮                                │
│                                                              │
│  [ 請選擇課程                                    ] [選單  ▾] │

│  ○  [ 選項 1                                      ] [×]      │

│                                                              │

│                      [複製] [刪除]  │  必填  [ ●── ]         │
└──────────────────────────────────────────────────────────────┘
```

互動：

- Enter：在目前選項後新增下一項。
- Backspace：空白選項再按一次可刪除。
- `×` 使用 inline SVG 的關閉 icon。
- 空選項、重複選項在該列下方顯示 inline error。
- 不用「刪除選項」文字按鈕，避免每列過度佔位。

## 七、進階設定與內部 key

Google Forms 不要求一般使用者編輯技術 key，因此預設收合。

### 7.1 收合

```text
[進階設定 ▾]
```

### 7.2 展開

```text
┌──────────────────────────────────────────────────────────────┐
│ 進階設定                                                     │
│                                                              │
│ 系統欄位代碼                                                 │
│ [ field_1                                                ]   │
│ 發佈後將用於資料欄位識別                                     │
└──────────────────────────────────────────────────────────────┘
```

建議規則：

- 新增題目時自動產生穩定 key，例如 `field_1`、`field_2`。
- key 不隨題目標題修改而改變。
- 管理者仍可在進階設定修改。
- 格式錯誤與重複 key 顯示 inline error。
- 已發佈表單維持唯讀。

> **待使用者確認：是否接受預設隱藏 key，改由系統自動產生？**

## 八、排序、複製與刪除

### 8.1 拖曳排序

```text
滑鼠移到 ⋮⋮
    │
    ├─ 按住拖曳
    ▼
題目卡跟隨移動
    │
    ├─ 插入位置顯示墨綠細線

```

拖曳中：

```text
        ───────────── 插入位置 ─────────────

┌──────────────────────────────────────────────────────────────┐
│ ⋮⋮  正在移動的題目卡                                        │
└──────────────────────────────────────────────────────────────┘
```

> 審查點：實作可先使用原生 pointer events，不預設引入拖曳套件。

### 8.2 複製

```text
[複製]
   │
   ├─ 在原題目下方建立副本
   ├─ 複製 label/type/options/required
   ├─ 產生新的 rowId
   └─ 產生新的唯一 key
```

### 8.3 刪除

```text
[刪除]
   │
   ▼
┌────────────────────────────────────┐
│ 刪除這個問題？                     │
│ 此操作無法復原。                   │
│                                    │
│       [取消]   [刪除問題]          │
└────────────────────────────────────┘
```

使用專案 Feedback `ConfirmDialog` danger variant；禁止瀏覽器原生 `confirm()`。

## 十一、預覽模式

### 11.1 編輯器切換

```text
[ 問題編輯 ]  [ 預覽 ]
```

使用 KJ Pill Tab 規範：

- 作用中：墨綠底、白字。
- 未作用：透明底、深炭灰字。
- 預覽不要求先儲存。

### 11.2 桌機預覽

```text
┌────────────────────────────────────────────────────────────────────────────────┐
│ [← 返回編輯]                  表單預覽                              [新視窗開啟] │
├────────────────────────────────────────────────────────────────────────────────┤
│                                                                                │
│                         ┌──────────────────────────────┐                       │
│                         │        [康九 Logo]          │                       │
│                         │                              │                       │
│                         │   康九冠軍課程調查           │                       │
│                         │                              │                       │
│                         │   姓名 *                     │                       │
│                         │  ┌────────────────────────┐  │                       │
│                         │  │ 搜尋或選擇姓名         │  │                       │
│                         │  └────────────────────────┘  │                       │
│                         │                              │                       │
│                         │   是否參加天驥加盟主 *       │                       │
│                         │  ┌──────────┐ ┌──────────┐  │                       │
│                         │  │    是    │ │    否    │  │                       │
│                         │  └──────────┘ └──────────┘  │                       │
│                         │                              │                       │
│                         │       [送出（預覽停用）]     │                       │
│                         └──────────────────────────────┘                       │
│                                                                                │
└────────────────────────────────────────────────────────────────────────────────┘
```

### 11.3 預覽規則

- 使用與 `SurveyFill` 相同的實際 renderer。
- 表單欄位可操作，以測試選單與必填體驗。
- 「送出」在未發佈草稿預覽中禁用，避免寫入資料。
- 預覽答案只存在本機 state。
- 返回編輯時不清除未儲存表單內容。
- 必填題標示 `*`，非必填不標示。

### 11.4 是否題的查看模式

編輯器骨架：

```text
○ 是
○ 否
```

實際填寫／預覽：

```text
┌──────────────────────┐  ┌──────────────────────┐
│          是          │  │          否          │
└──────────────────────┘  └──────────────────────┘
```

原因：管理者編輯時需要節省垂直空間；填寫者端仍保留現有大觸控區。


```text
┌──────────────────────────────────────────────────────────────┐
```
