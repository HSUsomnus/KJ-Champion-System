📖 LINE 整合測試說明

依你想在「Cursor 內」或「外部瀏覽器／LINE App」測試，方式不同。

1️⃣ 在 Cursor／本機瀏覽器測試（不需 LINE 登入）

  • 用 Cursor 內建瀏覽器或本機瀏覽器打開：
    http://localhost:8080/ 或 http://localhost:8080/?dev=1
  • 會自動進入開發模式，使用模擬 LINE 帳號，可測行事曆、列表、成員、個人、新增/編輯行程等。
  • 分享、關閉 LIFF 等需「真實 LINE 環境」的功能，在此模式下無法實際測到。
💡 Cursor 內建瀏覽器無法跑「真實 LINE」（LIFF 需在 LINE App 內或經 LINE OAuth 回傳）。要測 LINE 登入與分享，請用下面「外部瀏覽器／LINE」方式。

2️⃣ 在外部瀏覽器／LINE App 測 LINE 登入

  • 先把專案部署到對外可連的網址（例如 ngrok、或你已有的正式網域），且該網址需已填在 LINE Developers 的 LIFF Endpoint URL。
  • 方式 A：在手機 LINE 裡點你的 LIFF 連結（選單、 Rich Menu、訊息連結等），就會在 LINE 內開 APP，並自動完成登入。
  • 方式 B：在 Chrome／Edge 等外部瀏覽器直接打開同一個網址，並在網址加上 ?dev=0 關閉開發模式，會出現「使用 LINE 登入」按鈕，按下去會跳轉 LINE 登入，登入後再回到你的 APP。

用 ?dev=0 代表「強制使用真實 LINE」，不再用模擬帳號。

3️⃣ 模式切換一覽

  • ?dev=1 或 localhost → 開發模式，模擬帳號，適合在 Cursor／本機跑。
  • ?dev=0 → 關閉開發模式，走真實 LINE 登入，適合在外部瀏覽器或 LINE 內測試。

🐛 DEBUG：LINE 登入後出現「這個網站無法提供安全連線」或 ERR_SSL_PROTOCOL_ERROR

若登入完成後被導回時出現上述錯誤，通常是 LIFF 回傳網址用了 HTTPS，但本機只開 HTTP。

解法（本機測試時）：
到 LINE Developers Console → 你的 Channel → LIFF → 點你的 LIFF 應用 → 把 Endpoint URL 改成 http://localhost:8080（務必用 http，不要用 https）。
本機 npm run dev 只提供 HTTP，LINE 若設成 https://localhost:8080 登入後會導回 HTTPS，瀏覽器連不到就出現 ERR_SSL_PROTOCOL_ERROR。

改好後儲存，再重試「使用 LINE 登入」即可。
